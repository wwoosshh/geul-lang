### Task 2: `에`/`로` 조사 매개변수 이름 불일치 수정

**Files:**
- Modify: `자체호스팅/형태소분석기.글:651`

**원인:** `선언이름모드`에서 `에`/`로` 접사가 붙은 새 이름(예: `오른쪽에`)의 어근 `오른쪽`이 아직 심볼 테이블에 없으면 분리하지 않고 통째로 `오른쪽에`를 이름으로 등록. 함수 본문에서는 일반 모드로 `오른쪽` + `에`로 분리되어 이름 불일치 발생.

**수정 전략:** 651줄(`심볼인가` 검사) 뒤에, 어근이 미인식이고 조사 접사이며 어근이 2글자(6바이트) 이상이면 **분리하고 어근을 심볼 등록**하는 조건 추가. 이렇게 하면:
- `오른쪽에` → 어근 `오른쪽`(9바이트≥6) → 분리 ✓
- `가로` → 어근 `가`(3바이트<6) → 분리 안 함 ✓
- `세로` → 어근 `세`(3바이트<6) → 분리 안 함 ✓

- [ ] **Step 1: 형태소분석기.글에서 조건 추가**

651줄 뒤에 삽입:

```글
                        (* 선언 모드에서 조사 접사 + 어근 2글자 이상이면 분리 허용 *)
                        (어근알려짐 == 0) * 조사인가(선접종류) * (선어근길이 >= 6) > 0이면 {
                            어근알려짐 = 1.
                            심볼등록(선어근).
                        }
```

- [ ] **Step 2: 테스트 파일 작성 및 검증**

```글
포함 "std.gl"
[정수 왼쪽을 정수 오른쪽에 더하기]는 -> 정수 {
    정수 결과는 왼쪽 + 오른쪽.
    결과를 반환하다.
}
[시작하기]는 {
    정수 결과는 10을 20에 더하다.
    "10+20 = %d (기대:30)\n"을 결과를 쓰다.
}
```

기대 출력: `30`
