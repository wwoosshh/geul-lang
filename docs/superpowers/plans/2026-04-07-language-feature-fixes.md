# 언어 기능 크리티컬 이슈 해결 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 시스템 프로그래밍 언어로서의 크리티컬 이슈 6건 수정

**Architecture:** 토큰→렉서→파서→IR→코드젠 파이프라인 순서로 수정, 각 이슈 후 빌드 검증

---

## 이슈 목록 (구현 난이도 순)

| # | 이슈 | 난이도 | 수정 파일 |
|---|------|--------|-----------|
| 1 | 비트 복합 대입 `&= \|= ^= <<= >>=` | 쉬움 | tokens, lexer, parser, ir_transform |
| 2 | for 루프 증감부 검증 (P1-3) | 쉬움 | ir_transform |
| 3 | 정수 배열 원소 크기 (P1-2) | 쉬움 | ir_transform |
| 4 | 타입 오류 빌드 중단 옵션 | 쉬움 | compiler |
| 5 | 함수 포인터 | 어려움 | tokens, parser, ir, ir_transform, pe_gen |
| 6 | --help CLI 도움말 | 쉬움 | compiler |

  종합 판정

  시스템 프로그래밍 기능 달성도:

    타입 시스템     ████████░░  80%  (unsigned/함수포인터/union 네이티브
   부재)
    메모리 제어     ███████░░░  70%  (포인터 산술 바이트 단위, 구조체 8B
   고정)
    비트 연산       ████████░░  80%  (비트 복합 대입 없음)
    제어 흐름       ████████░░  80%  (for 증감 미완, goto 없음)
    함수 시스템     ███████░░░  70%  (함수 포인터 없음, va_list 네이티브
   없음)
    빌드 시스템     █████░░░░░  50%  (단일 파일, 디버그 정보 없음)
    최적화         ███████░░░  70%  (LICM/인라인 비활성)
    크로스 플랫폼   ███░░░░░░░  30%  (Windows x86-64 전용)
    ─────────────────────────────────
    종합           ██████░░░░  66%

  현재 위치: 자체호스팅 가능한 단일 플랫폼 컴파일러. 기본 기능은
  안정적이나, 시스템 프로그래밍 언어로서의 핵심 기능(함수 포인터,
  unsigned, union, 포인터 산술, 다중 파일)이 부족.
