## 미해결 이슈 — 우선순위별

### P0: 크리티컬 (시스템 언어 필수)

| # | 이슈 | 난이도 | 수정 영역 | 상세 |
|---|------|--------|-----------|------|
| A1 | **unsigned 산술** | 중간 | pe_gen | `부호없는` 타입이 파싱되지만 모든 산술이 signed 명령(imul/idiv) 사용. 비트 시프트, 비교, 나누기에서 결과 오류 가능. `mul`/`div`/`shr` vs `imul`/`idiv`/`sar` 분기 필요 |
| A2 | **union 네이티브 지원** | 중간 | ir_transform, pe_gen | `합침` 선언이 파싱/AST 생성되지만 IR 변환과 네이티브 코드젠에서 무시됨. C 백엔드에서만 동작 |
| A3 | **typedef 네이티브 지원** | 쉬움 | ir_transform | `별칭` 선언이 파싱되지만 네이티브 백엔드에서 타입 해석 시 별칭을 원본 타입으로 치환하지 않음 |
| A4 | **가변 인자 (va_list) 네이티브** | 어려움 | ir, ir_transform, pe_gen | `가변목록/가변시작/가변끝/가변인자` 토큰 존재, C 백엔드에서 va_list로 매핑. 네이티브에서 Windows x64 va_list ABI 구현 필요 |
| A5 | **포인터 산술 타입 인식** | 중간 | ir_transform | `정수 참조 ptr`에 `ptr + 1`이 1바이트가 아닌 `sizeof(정수)` 바이트만큼 이동해야 함. 현재 바이트 단위로만 동작 |

### P1: 높음 (실용적 프로그래밍 필수)

| # | 이슈 | 난이도 | 수정 영역 | 상세 |
|---|------|--------|-----------|------|
| B1 | **구조체 필드 가변 크기** | 어려움 | ir_transform, pe_gen | 모든 필드 8B 고정. `문자`(1B), `짧은정수`(2B) 필드도 8B 차지. C 구조체 호환 불가. 타입별 크기 테이블 + 정렬 로직 필요 |
| B2 | **static 저장 클래스 네이티브** | 쉬움 | pe_gen | `정적` 플래그가 AST에 설정되지만 네이티브 백엔드에서 BSS/data 섹션 배치 없이 일반 지역 변수로 처리 |
| B3 | **const 강제** | 쉬움 | ir_transform | `상수` 플래그가 AST에 설정되지만 수정 시 오류를 발생시키지 않음 |
| B4 | **LICM 최적화 활성화** | 중간 | pe_gen | 루프 불변 코드 이동이 구현되어 있으나 정확성 버그로 비활성화 상태 |
| B5 | **일반 인라이닝 활성화** | 중간 | pe_gen | 소함수 인라이닝이 구현되어 있으나 대형 프로그램 범위 밖 참조 버그로 비활성화 |

### P2: 중간 (언어 완성도)

| # | 이슈 | 난이도 | 수정 영역 | 상세 |
|---|------|--------|-----------|------|
| C1 | **goto/label** | 중간 | parser, ir_transform, pe_gen | AST 노드 13/14 정의됨, 파서/코드젠 없음. 상태 머신, 에러 복구 패턴에 필요 |
| C2 | **range-for** `부터/까지` | 중간 | parser, ir_transform | AST 노드 10 정의됨, 파서 없음. `(1부터 10까지)동안 { }` 반복 편의성 |

| C4 | **매개변수 매크로** | 중간 | lexer, compiler | 현재 `정의 이름 값` 상수 매크로만. `정의 MAX(a,b) ...` 형태 미지원 |
| C5 | **에러 처리** `없으면/실패면/정리` | 중간 | parser, ir_transform, pe_gen | 설계 완료 (docs/20-에러처리-설계.md), 미구현 |
| C6 | **한글 어미 확장** `-서/-지만/-도록/-며` | 쉬움 | parser | 토큰 정의됨, 렉서 분리됨, 파서에서 무시 |

## 다음 작업 권장 순서

**즉시 (쉬운 승리):**
1. **A3 typedef** — `별칭` 타입을 원본으로 치환하는 로직 추가 (ir_transform.gl ~50줄)
2. **B3 const 강제** — `상수` 변수 대입 시 오류 발생 (~10줄)
3. **B2 static** — BSS 섹션에 정적 변수 배치 (pe_gen.gl ~30줄)

**단기 (높은 임팩트):**
4. **A1 unsigned** — `mul`/`div`/`shr` 명령 분기 (pe_gen.gl ~100줄)
5. **A5 포인터 산술** — 원소 크기 곱셈 삽입 (ir_transform.gl ~30줄)
6. **A2 union** — 최대 필드 크기 기반 메모리 할당 (ir_transform.gl + pe_gen.gl ~80줄)

**중기:**
7. **C1 goto/label** — 파서+IR+코드젠 (~150줄)
8. **C2 range-for** — 파서 디슈가링 (~50줄)
9. **C5 에러 처리** — 설계 문서 기반 구현 (~200줄)
10. **A4 va_list** — Windows x64 ABI (~200줄)
