# v0.5 릴리즈 — 네이티브 자체호스팅 배포

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 네이티브 자체호스팅 컴파일러 v0.5를 정리·문서화·배포한다.

**Architecture:** 최신 자체호스팅 컴파일러를 빌드하고, 배포 폴더를 갱신하고, 버전 관리 체계와 문법 레퍼런스를 작성한 뒤 GitHub에 푸시하고 릴리즈를 생성한다.

**Tech Stack:** 글 언어, PE 네이티브 컴파일러, GitHub CLI (gh)

### Task 1: 최신 자체호스팅 컴파일러 빌드 및 배포 폴더 갱신

**Files:**
- Modify: `배포/네이티브컴파일러.exe` (교체)
- Modify: `배포/std.gl` (이미 갱신됨)
- Delete: `배포/nc_stage0.exe` (불필요)
- Delete: `예제/*.exe` (빌드 산출물)

- [ ] **Step 1: 최신 stage0으로 stage1 빌드**

```bash
cd "C:/workspace/한글"
"자체호스팅/네이티브컴파일러.exe" "자체호스팅/네이티브컴파일러.글" -o "예제/stage1.exe"
```

- [ ] **Step 2: stage1으로 stage2 빌드 (ASCII 경로)**

```bash
cp 자체호스팅/*.글 /c/temp/geul/
cp 배포/std.gl /c/temp/geul/
cp 예제/stage1.exe /c/temp/geul/
cd /c/temp/geul
./stage1.exe 네이티브컴파일러.글 -o stage2.exe
```

- [ ] **Step 3: stage2로 stage3 빌드 — 고정점 검증**

```bash
cd /c/temp/geul
./stage2.exe 네이티브컴파일러.글 -o stage3.exe
sha256sum stage2.exe stage3.exe
```
Expected: 두 SHA256 일치

- [ ] **Step 4: 배포 폴더 갱신**

```bash
cd "C:/workspace/한글"
cp /c/temp/geul/stage3.exe 배포/네이티브컴파일러.exe
rm -f 배포/nc_stage0.exe
rm -f 예제/*.exe
```

- [ ] **Step 5: 배포 컴파일러 최종 검증**

```bash
cd "C:/workspace/한글"
"배포/네이티브컴파일러.exe" "예제/간단.글" -o "예제/간단.exe" && "예제/간단.exe"
"배포/네이티브컴파일러.exe" "예제/종합검증.글" -o "예제/종합검증.exe" && "예제/종합검증.exe"
rm -f 예제/*.exe
```
Expected: "안녕 세계", "모든 테스트 통과"

- [ ] **Step 6: 커밋**

```bash
git add 배포/ 예제/
git commit -m "release: v0.5 네이티브 자체호스팅 컴파일러 배포 갱신"
```

### Task 3: 문법 레퍼런스 (v0.5) 작성

**Files:**
- Create: `docs/글-문법-레퍼런스-v0.5.md`

현재 네이티브 컴파일러에서 **실제 동작이 검증된 문법만** 포함한다. 각 항목에 코드 예제를 반드시 첨부한다.

- [ ] **Step 1: 문법 레퍼런스 작성**

아래 섹션을 포함:
1. 기본 구조 (프로그램 진입점, 문장 종결, 주석)
2. 타입 (정수, 실수, 문자열, 문자, 참거짓, 공허)
3. 변수 선언과 대입
4. 함수 정의와 호출 (SOV 동사형 + 직접 호출)
5. 제어 흐름 (이면/아니면, 동안, 반복, 갈래, 반복하기~동안)
6. 연산자 (산술, 비교, 논리, 복합 대입, 증감, 삼항)
7. 배열
8. 구조체(묶음), 열거형(나열), 공용체(합침)
9. 포인터와 메모리
10. 전처리기 (포함, 정의, 만약정의)
11. 고급 기능 (별칭, 외부, 정적, 상수, 함수포인터, 가변인자)
12. 표준 라이브러리 (std.gl)
13. 네이티브 컴파일 사용법

- [ ] **Step 2: 예제 코드가 실제 컴파일되는지 확인**

문서의 주요 예제를 stage3 컴파일러로 컴파일하여 검증.

- [ ] **Step 3: 커밋**

```bash
git add docs/글-문법-레퍼런스-v0.5.md
git commit -m "docs: v0.5 문법 레퍼런스 작성"
```

### Task 5: GitHub 릴리즈 배포

- [ ] **Step 1: 릴리즈 생성**

```bash
gh release create v0.5.0 \
  "배포/네이티브컴파일러.exe" \
  "배포/std.gl" \
  --title "v0.5.0 — 네이티브 자체호스팅" \
  --notes "릴리즈 노트 내용"
```

릴리즈 노트에 포함할 내용:
- 네이티브 자체호스팅 달성 (고정점 SHA256)
- 수정된 버그 목록
- 사용법 안내
- 성능 수치
