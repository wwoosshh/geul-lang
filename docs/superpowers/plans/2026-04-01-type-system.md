# 글 네이티브 컴파일러 타입 시스템 구축 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 네이티브 컴파일러에 정적 타입 검사를 추가하여, 타입 불일치를 컴파일 시점에 에러로 잡아냄

**Architecture:** IR변환기에서 표현식 평가 시 타입을 추적(전파)하고, 대입·호출·반환 등 경계 지점에서 타입 호환성을 검사. 기존 AST 타입 노드 인프라를 활용.

**Tech Stack:** 글 언어 (자체호스팅 소스), 글cc (재컴파일용)

---

## 현재 상태 요약

| 계층 | 상태 | 설명 |
|------|------|------|
| AST | ✅ 완성 | 5가지 타입 노드, 매개변수·반환타입 저장됨 |
| IR | ⚠️ 부분 | `임시변수타입풀[]` 존재하지만 미활용 |
| 변수 | ⚠️ 부분 | `변수타입종류풀[]` 보간 서식용으로만 사용 |
| 함수 | ⚠️ 부분 | `중간함수반환타입풀[]` 저장만, 검증 없음 |
| 검사 | ❌ 없음 | 어떤 지점에서도 타입 호환성 검증 안 함 |

## 핵심 설계 결정

### 타입 표현
기존 토큰 종류를 타입 ID로 재활용:
- `종류_정수`(20), `종류_긴정수`(21), `종류_실수`(25), `종류_문자`(27), `종류_문자열`(28), `종류_참거짓`(29), `종류_공허`(30)
- 포인터: `100 + 기본타입` (예: 정수 포인터 = 120)
- 묶음: `200 + 묶음인덱스`
- 알 수 없음: `-1`

### 타입 호환성 규칙
```
정수 ↔ 긴정수 ↔ 짧은정수 ↔ 작은정수 ↔ 부호없는 ↔ 참거짓   (정수 계열 호환)
실수 ↔ 짧은실수                                              (실수 계열 호환)
정수계열 → 실수계열                                           (암시적 확대)
문자 ↔ 정수계열                                              (C 호환)
문자열 ↔ 참조(문자)                                          (포인터 호환)
그 외                                                        (불일치 → 에러)
```

### 에러 정책
- **에러 (컴파일 중단)**: 정수 ← 문자열, 문자열 ← 정수, 묶음 ← 정수 등 명백한 불일치
- **경고 (계속 진행)**: 정수 ← 실수 (축소 변환), 긴정수 ← 정수 (확대)
- **허용**: 같은 계열 내 변환, 형변환 연산자 사용 시

현재 `표현식변환`은 임시변수 인덱스만 반환. 각 임시변수의 타입을 `임시변수타입풀[]`에 저장하도록 변경.

- [ ] **Step 1: 타입 호환성 판별 함수 추가**

IR변환기.글 상단(함수 정의 영역)에 추가:

```글
(* ===== 타입 시스템 ===== *)

(* 정수 계열인지 판별 *)
[정수 타입을 정수계열인가]는 → 참거짓 {
    (타입 == 종류_정수) + (타입 == 종류_긴정수) + (타입 == 종류_짧은정수) + (타입 == 종류_작은정수) + (타입 == 종류_부호없는) + (타입 == 종류_참거짓) + (타입 == 종류_문자) > 0이면 { 반환 1. }
    반환 0.
}

(* 실수 계열인지 판별 *)
[정수 타입을 실수계열인가]는 → 참거짓 {
    (타입 == 종류_실수) + (타입 == 종류_짧은실수) > 0이면 { 반환 1. }
    반환 0.
}

(* 두 타입이 호환되는지 판별: 0=불일치, 1=호환, 2=축소경고 *)
[정수 원본타입을 정수 대상타입으로 타입호환검사]는 → 정수 {
    원본타입 == 대상타입이면 { 반환 1. }
    원본타입 < 0이면 { 반환 1. }  (* 알 수 없음 → 통과 *)
    대상타입 < 0이면 { 반환 1. }
    (* 정수 계열 내 호환 *)
    원본타입을 정수계열인가다 * 대상타입을 정수계열인가다 > 0이면 { 반환 1. }
    (* 실수 계열 내 호환 *)
    원본타입을 실수계열인가다 * 대상타입을 실수계열인가다 > 0이면 { 반환 1. }
    (* 정수 → 실수 암시적 확대 *)
    원본타입을 정수계열인가다 * 대상타입을 실수계열인가다 > 0이면 { 반환 1. }
    (* 실수 → 정수 축소 경고 *)
    원본타입을 실수계열인가다 * 대상타입을 정수계열인가다 > 0이면 { 반환 2. }
    (* 문자열 ↔ 포인터 호환 *)
    (원본타입 == 종류_문자열) * (대상타입 >= 100) > 0이면 { 반환 1. }
    (대상타입 == 종류_문자열) * (원본타입 >= 100) > 0이면 { 반환 1. }
    (* 그 외 불일치 *)
    반환 0.
}
```

- [ ] **Step 2: 1차 패스 — 함수 시그니처 수집**

`프로그램변환`에서 기존 1차 패스(묶음/나열 등록) 직후에 함수 정의들을 순회하여 시그니처 등록:

```글
(* 함수 시그니처 수집 *)
(노드종류풀[선언] == 노드_함수정의)이면 {
    함수시그이름풀[함수시그수] = 노드이름풀[선언].
    (* 반환 타입 *)
    정수 반환노드 = 노드추가풀[선언].
    반환노드 >= 0이면 {
        함수시그반환타입풀[함수시그수] = 노드연산자풀[반환노드].
    } 아니면 {
        함수시그반환타입풀[함수시그수] = 종류_공허.
    }
    (* 매개변수 타입 수집 *)
    함수시그매개시작풀[함수시그수] = 함수시그매개총수.
    정수 매개 = 노드왼쪽풀[선언].
    정수 매개수 = 0.
    (매개 >= 0)동안 {
        정수 매개타입노드 = 매개타입풀[매개].
        (매개타입노드 >= 0) * (노드종류풀[매개타입노드] == 노드_기본타입) > 0이면 {
            함수시그매개타입풀[함수시그매개총수] = 노드연산자풀[매개타입노드].
        } 아니면 {
            함수시그매개타입풀[함수시그매개총수] = -1.
        }
        함수시그매개총수를 증가하다.
        매개수를 증가하다.
        매개 = 매개다음풀[매개].
    }
    함수시그매개수풀[함수시그수] = 매개수.
    함수시그수를 증가하다.
}
```

            쓰기("타입 오류: 변수 '%s' — %s 타입에 %s 값을 대입할 수 없습니다\n",

        호환 == 2이면 {
            쓰기("경고: 변수 '%s' — 축소 변환 (%s → %s)\n",
                이름, 토큰이름(초기타입), 토큰이름(선언타입)).
        }
    }
    정수 값 = 초기노드를 표현식변환다.
    변수를 값을 복사명령생성다.
}

## 위험 요소

1. **기존 코드 깨짐**: 글 표준 라이브러리나 자체호스팅 소스에 암시적 타입 변환이 존재할 수 있음. 타입 검사를 추가하면 기존 코드가 컴파일 안 될 수 있음 → **자체호스팅 소스를 먼저 검증해야 함**

2. **임시변수 타입 전파 누락**: 모든 표현식 경로에서 타입을 정확히 전파해야 함. 누락 시 `-1`(알 수 없음)으로 처리하여 오탐 방지

3. **외부 C 함수**: `printf`, `malloc` 등 C 함수는 시그니처가 없음 → 시그니처 미등록 함수는 검사 건너뛰기

4. **매크로(`정의`)를 통한 함수 별칭**: `정의 쓰기 printf` 같은 매크로는 시그니처 테이블에 없음 → 미등록 함수 허용 정책 필요
