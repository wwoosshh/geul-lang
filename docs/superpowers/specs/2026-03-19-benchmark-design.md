# 글 언어 종합 벤치마크 시스템 설계

## 1. 개요

### 목적
- 글 언어의 **컴파일러 성능**과 **런타임 성능**을 객관적으로 측정
- C, Python, Go, Rust, Java 5개 대중 언어와 비교하여 강점/약점 식별
- 개발자에게 어필할 수 있는 성능 데이터 확보

### 접근법: 하이브리드 측정 (내부 + 외부)
- **내부 측정**: 각 프로그램 내부에서 고정밀 타이머로 핵심 연산만 측정
- **외부 측정**: PowerShell에서 전체 프로세스 실행 시간 측정
- **분리 가능한 데이터**: 컴파일 시간 / 프로세스 시작 시간 / 순수 연산 시간

## 4. 측정 방식

### 4.1 내부 타이머 (프로그램 내부)

각 언어별 고정밀 타이머:

| 언어 | 타이머 API | 정밀도 |
|------|-----------|--------|
| 글 | QueryPerformanceCounter (신규 추가) | 마이크로초 |
| C | QueryPerformanceCounter | 마이크로초 |
| Python | time.perf_counter() | 마이크로초 |
| Go | time.Now() / time.Since() | 나노초 |
| Rust | std::time::Instant | 나노초 |
| Java | System.nanoTime() | 나노초 |

모든 프로그램은 핵심 연산 구간만 감싸서 측정 후 표준 출력:
```
RESULT:123.456
```
단위: 밀리초(ms), 소수점 3자리.

### 4.2 외부 타이머 (PowerShell)

```powershell
$sw = [System.Diagnostics.Stopwatch]::StartNew()
# 프로세스 실행
$sw.Stop()
$외부시간 = $sw.Elapsed.TotalMilliseconds
```

### 4.3 실행 프로토콜

1. 각 벤치마크 **5회 반복** 실행
2. 1회차 제외 (워밍업 — 주로 Java JIT, 파일시스템 캐시 효과), 2~5회차 사용
3. 중앙값 산출: 4개 값 정렬 후 2번째와 3번째의 평균 (하위 중앙값)
4. 컴파일 시간은 별도 1회 측정
5. 결과: 내부 시간(중앙값), 외부 시간(중앙값), C 대비 배율

## 5. 글 표준라이브러리 확장

### 신규 모듈: `표준/정밀시간.글`

Windows `QueryPerformanceCounter`/`QueryPerformanceFrequency` 래핑.

> **참고**: 생성된 C 코드에 `#ifdef _WIN32`로 `<windows.h>`가 이미 포함됨.
> `kernel32.lib`는 MSVC 기본 링크 대상이므로 별도 설정 불필요.
> `LARGE_INTEGER`는 `long long`과 ABI 호환이므로 `긴정수 참조`로 선언.

```글
(* 정밀 시간 측정 라이브러리 — Windows QueryPerformanceCounter 래핑 *)

외부 [긴정수 참조 값을 QueryPerformanceCounter]는 -> 정수.
외부 [긴정수 참조 값을 QueryPerformanceFrequency]는 -> 정수.

정적 긴정수 시작틱 = 0.
정적 긴정수 측정빈도 = 0.

[정밀시작]는 {
    긴정수 빈도 = 0.
    &빈도를 QueryPerformanceFrequency다.
    측정빈도 = 빈도다.
    &시작틱을 QueryPerformanceCounter다.
}

[정밀끝]는 -> 실수 {
    긴정수 끝틱 = 0.
    &끝틱을 QueryPerformanceCounter다.
    반환 (끝틱 - 시작틱) * 1000.0 / 측정빈도.
}
```

## 7. 비교 대상 언어 및 컴파일 옵션

| 언어 | 컴파일러 | 최적화 플래그 | 비고 |
|------|---------|-------------|------|
| C | cl.exe (MSVC) | /O2 | 기준선 |
| 글 | 글도구.exe → cl.exe | /O2 (C 단계) | 트랜스파일 오버헤드 측정 |
| Python | python.exe | - | 인터프리터 |
| Go | go build | 기본 | 기본 최적화 |
| Rust | rustc | -C opt-level=2 | release 모드 |
| Java | javac + java | 기본 (JDK 17+ -server 기본값) | JIT 최적화 |

## 8. 벤치마크 셋업 (파일 I/O용)

`실행.ps1`의 초기화 단계에서 10MB 테스트 파일 자동 생성:
- 파일명: `벤치마크/결과/testdata.txt`
- 내용: 영문 Lorem ipsum 반복 (언어간 공정 비교를 위해 ASCII 텍스트)
- 공백으로 구분된 단어 나열, 줄바꿈 포함
- 벤치마크 실행 전 존재 여부 확인, 없으면 자동 생성

## 9. 전제 조건

### 필수 설치
- MSVC (cl.exe) — 이미 설치됨
- Python 3.x
- Go 1.20+
- Rust (rustc)
- Java JDK 17+

### 선택적
- 없는 언어는 자동 스킵, 설치된 언어만 비교

### Go 벤치마크 빌드 방식
- 각 `.go` 파일은 `package main` 단독 파일
- `go build -o output.exe file.go`로 개별 빌드 (go.mod 불필요)

## 10. 구현 순서

1. `표준/정밀시간.글` 모듈 추가
2. 글 벤치마크 6개 작성
3. C 벤치마크 6개 작성 (기준선)
4. Python/Go/Rust/Java 벤치마크 각 6개
5. `설정.ps1` — 언어 탐지, 경로 설정
6. `실행.ps1` — 중앙 제어, 컴파일/실행/측정/출력
7. 컴파일러 스트레스 테스트 2개 (글 전용)
8. 종합 요약 출력 로직
