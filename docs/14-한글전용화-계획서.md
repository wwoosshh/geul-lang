## 1. 현재 상태 진단

### 1.1 달성한 것

| 항목 | 상태 |
|------|------|
| 컴파일러 소스코드 100% 글 언어 | **완료** |
| 자체호스팅 (자기 자신 컴파일) | **완료** (고정점 도달, SHA256 일치) |
| 기본 타입 한글화 (정수, 문자열, 참거짓 등) | **완료** |
| 제어흐름 한글화 (이면, 동안, 반환 등) | **완료** |
| 조사/어미 기반 문법 | **완료** |
| 표준 라이브러리 한글 래퍼 (4개 모듈) | **기초 완료** |

### 1.2 남은 C 의존성

#### 의존성 A: 실행 파이프라인
```
.글 소스 → [글cc] → .c 파일 → [cl.exe (MSVC)] → .exe
                      ^^^^^^^^    ^^^^^^^^^^^^^^^
                      C 코드 생성   C 컴파일러 필요
```
글cc는 트랜스파일러이며, 최종 기계어 생성은 MSVC가 담당한다.

#### 의존성 B: C 표준 라이브러리 함수 직접 노출
사용자 코드에서 영어 함수명이 그대로 노출된다:
```글
(* 현재: 영어 함수명 직접 사용 *)
"결과: %d\n"을 값을 printf다.
문자열 버퍼는 malloc(1024).
strcmp(가, 나) == 0이면 { ... }
```

#### 의존성 C: 컴파일러 내부도 C 함수 직접 사용
컴파일러 자체 소스(7개 파일)에서 사용하는 C 함수:
- `printf`, `fprintf`, `snprintf` — 출력
- `malloc`, `free`, `realloc` — 메모리
- `strcmp`, `strncmp`, `strlen`, `strdup`, `strncpy` — 문자열
- `fopen`, `fclose`, `fread`, `fseek`, `ftell` — 파일
- `system`, `remove`, `atoi` — 체계
- `freopen` — 출력 리다이렉트
- `memset`, `memcpy` — 메모리 조작

## 2. 한글 전용화의 목표

### 최종 비전
```
다른 개발자가 글 언어로 코딩할 때:
 - 영어 함수명을 하나도 볼 필요가 없다
 - 포함 "표준/입출력.글" 한 줄로 한글 함수를 사용한다
 - 타입, 제어흐름, 함수, 라이브러리 모두 한글이다
```

### 목표 코드 예시 (최종 형태)
```글
포함 "표준/입출력.글"
포함 "표준/문자열.글"
포함 "표준/메모리.글"

[시작하기]는 {
    문자열 이름은 "세계".
    "안녕하세요, %s!\n"을 이름을 쓰기다.

    정수 크기는 100.
    공허 참조 버퍼는 크기를 할당다.
    버퍼를 해제다.

    문자열 가는 "한글".
    문자열 나는 "한글".
    가를 나와 문자열_같은가 이면 {
        "같습니다\n"을 쓰기다.
    }
}
```

## 3. 단계별 실행 계획

### 3단계 구조

```
[1단계] 표준 라이브러리 완성        ← 사용자가 한글만으로 코딩 가능
           ↓
[2단계] 컴파일러 내부 한글화        ← 컴파일러 자체도 영어 함수 제거
           ↓
[3단계] C 트랜스파일 독립           ← 장기: 네이티브 코드 직접 생성
```

### 1단계: 표준 라이브러리 완성 (사용자 한글 전용화)

> 목표: `포함 "표준/입출력.글"` 하면 영어 함수 없이 코딩 가능

#### 1-1. 기존 모듈 점검 및 보강

| 모듈 | 파일 | 현재 함수 | 추가 필요 |
|------|------|-----------|-----------|
| 입출력 | 표준/입출력.글 | 쓰기(printf), 파일_열기, 파일_닫기, 파일_읽기, 파일_쓰기, 줄_읽기, 쓰기_줄 등 | 서식쓰기(fprintf→파일), 오류쓰기(stderr) |
| 문자열 | 표준/문자열.글 | 글자수, 문자열_비교, 문자열_같은가, 문자열_복사, 문자열_연결, 문자열_찾기, 문자열_복제, 정수로_변환 등 | 문자열_서식(snprintf 래퍼) |
| 메모리 | 표준/메모리.글 | 할당, 초기할당, 재할당, 해제 | 메모리_복사, 메모리_채우기 (문자열.글에 있음→분리) |
| 체계 | 표준/체계.글 | 종료, 명령_실행, 절댓값 | — |

#### 1-2. 신규 모듈 작성

| 모듈 | 파일 | 한글 함수 | C 함수 래핑 |
|------|------|-----------|-------------|
| 수학 | 표준/수학.글 | 제곱근, 거듭제곱, 반올림, 올림, 내림, 나머지, 최솟값, 최댓값 | sqrt, pow, round, ceil, floor, fmod, fmin, fmax |
| 변환 | 표준/변환.글 | 정수를_글로, 실수를_글로, 글을_정수로, 글을_실수로 | sprintf, atoi, atof, strtol, strtod |
| 시간 | 표준/시간.글 | 현재시각, 시간차이 | time, difftime, clock |

#### 1-3. 통합 표준 모듈 작성

하나만 포함하면 기본 기능 전부 사용 가능한 편의 모듈:

```글
(* 표준/기본.글 — 통합 모듈 *)
포함 "표준/입출력.글"
포함 "표준/문자열.글"
포함 "표준/메모리.글"
포함 "표준/체계.글"
```

사용자는 `포함 "표준/기본.글"` 한 줄로 모든 기본 한글 함수를 사용한다.

#### 1-4. 검증 기준
- [x] `포함 "표준/기본.글"` 후 영어 함수 없이 기본 프로그램 작성 가능 ✓ (2026-03-17)
- [x] Hello World, 파일 읽기/쓰기, 문자열 처리, 동적 메모리 할당 모두 한글 함수로 동작 ✓
- [x] 예제 프로그램 5개 이상 작성하여 검증 ✓ (한글전용_인사/문자열/메모리/파일처리/수학)

### 2단계: 컴파일러 내부 한글화

> 목표: 컴파일러 소스(자체호스팅)에서도 C 함수명 제거

#### 2-1. 컴파일러 내부 C 함수 사용 현황

자체호스팅 소스 7개 파일에서 직접 호출하는 C 함수:

| C 함수 | 호출 횟수(대략) | 대체 한글 함수 |
|--------|----------------|----------------|
| printf | ~200회 | 쓰기 |
| fprintf | ~10회 | 오류쓰기 (stderr용) |
| snprintf | ~15회 | 서식쓰기 (버퍼용) |
| malloc | ~30회 | 할당 |
| free | ~20회 | 해제 |
| strcmp | ~40회 | 문자열_비교 |
| strncmp | ~30회 | 문자열_일부비교 |
| strlen | ~20회 | 글자수 |
| strdup | ~10회 | 문자열_복제 |
| strncpy | ~10회 | 문자열_일부복사 |
| memset | ~5회 | 메모리_채우기 |
| fopen | ~5회 | 파일_열기 |
| fclose | ~3회 | 파일_닫기 |
| fread | ~3회 | 파일_읽기 |
| fseek | ~3회 | 파일_위치이동 |
| ftell | ~2회 | 파일_위치 |
| freopen | ~2회 | (신규 필요) 출력_전환 |
| system | ~1회 | 명령_실행 |
| remove | ~1회 | (신규 필요) 파일_삭제 |
| atoi | ~1회 | 정수로_변환 |

#### 2-2. 단계적 교체 전략

**원칙: 한 모듈씩 교체하고 매번 자체호스팅 검증**

```
교체 순서:
1. 주.글         — 메인 진입점 (가장 단순)
2. 구문트리.글    — AST 헬퍼
3. 토큰정의.글    — 토큰 유틸리티
4. 유니코드.글    — 유니코드 헬퍼
5. 형태소분석기.글 — 렉서 (가장 복잡)
6. 구문분석기.글   — 파서
7. 코드생성기.글   — 코드 생성 (printf 가장 많음)
```

각 교체 후:
1. 부트스트랩으로 빌드
2. 자체 컴파일 테스트
3. 고정점 도달 확인

#### 2-3. 검증 기준
- [x] 자체호스팅 소스 6개 파일에서 C 함수명 직접 호출 0건 ✓ (2026-03-17)
- [x] 모든 C 함수 호출이 `정의` 매크로를 통한 한글 래퍼로 대체 ✓
- [x] 자체호스팅 고정점 유지 ✓ (SHA256: `47367982ca39f7deb883d266e7a9c9bcf9360629f5e5486eb5a331919e28abad`)

#### 2-4. 추가 구현 사항 (2026-03-17)
- 구문분석기에 `정의`(#define) 파싱 추가 (`노드_매크로정의`)
- 구문분석기에 `포함`(#include) 건너뛰기 추가
- 코드생성기에 `노드_매크로정의` → `#define` 출력 추가
- `정의` 매크로 전략: 주.글 상단에 30+ 매핑으로 모든 포함 파일에 자동 전파

### 3단계: C 트랜스파일 독립 (장기 목표)

> 목표: cl.exe 없이 .글 → .exe 직접 생성

이 단계는 12-독립컴파일러-종합로드맵.md의 P3~P6에 해당한다.
한글 전용화(1~2단계)와 독립적으로 진행 가능하지만, 우선순위는 1~2단계가 높다.

#### 3-1. 중간 경로: C 백엔드 유지 + 한글 전용
당분간 C 트랜스파일 방식을 유지하되, **사용자에게는 C가 보이지 않도록** 한다:
- 생성된 .c 파일은 자동 삭제 (현재 구현됨)
- 컴파일 과정은 글cc가 자동 관리
- 사용자는 `.글 → .exe` 과정만 인식

#### 3-2. 장기 경로: 네이티브 백엔드
12-독립컴파일러-종합로드맵.md의 P3~P6 경로를 따라 진행:
- P3: 중간표현(IR) 도입
- P4: x86-64 네이티브 코드 생성
- P5: 자체호스팅을 네이티브 백엔드로 전환
- P6: 자체 어셈블러 + 링커 + 런타임

## 5. 설계 원칙

### 5.1 명명 규칙

| 분류 | 규칙 | 예시 |
|------|------|------|
| 기본 타입 | 순한글 | 정수, 문자열, 참거짓, 실수 |
| 제어 키워드 | 한글 어미/조사 | 이면, 동안, 반환, 탈출 |
| 표준 함수 | 동사형 한글 | 할당, 해제, 쓰기, 글자수 |
| 복합 함수 | 밑줄 구분 | 파일_열기, 문자열_비교 |
| 상수/나열 | 밑줄 접두 | 종류_정수, 종류_문자열 |

### 5.2 외부 함수 격리 원칙

```
사용자 코드 계층:  한글 함수만 사용 (쓰기, 할당, 글자수 등)
          ↓
표준 라이브러리:   한글 → C 변환 래퍼 (내부에서만 extern 사용)
          ↓
C 표준 라이브러리: printf, malloc, strlen (사용자에게 비노출)
```

영어 함수명은 **오직 표준 라이브러리 내부의 `외부` 선언에서만** 존재하며,
사용자 코드에는 절대 노출되지 않는다.

### 5.3 하위 호환

- 기존 C 함수 직접 호출 코드도 계속 동작 (금지하지 않음)
- 한글 래퍼는 **권장**, C 함수 직접 사용은 **비권장**
- 문서와 예제는 모두 한글 함수만 사용
