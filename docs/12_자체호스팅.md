# 12. 자체호스팅 (Self-Hosting) — 완료 (2026-03-17)

## 개요

글 컴파일러의 자체호스팅이 **완료**되었다.
글 언어로 작성된 컴파일러가 **자기 자신의 소스코드를 컴파일**하여 동일한 바이너리를 생성한다.

```
단계 0: 부트스트랩    글cc.exe + 자체호스팅/*.글 → 주_stage0.exe
단계 1: 교차 컴파일    주_stage0.exe + 자체호스팅/*.글 → 주_stage1.exe
단계 2: 자체 컴파일    주_stage1.exe + 자체호스팅/*.글 → 주_stage2.exe
검증:   SHA256(Stage1) == SHA256(Stage2) ✓
```

**고정점 달성:** `0C122A963AD3E951E919D9B1D0687430E1B93511506C25B23874BA0FC96385A9`

## 아키텍처

### 컴파일 파이프라인

```
소스.글 → [전처리] → [렉서] → [파서] → [AST] → [코드생성기] → 소스.c → [cl.exe] → 소스.exe
          포함 처리   형태소분리  SOV파싱   SoA풀    C11 출력
```

### SoA (Struct-of-Arrays) 패턴

글 언어는 전역 구조체 변수를 지원하지 않으므로,
모든 구조체 데이터를 **병렬 배열**로 표현한다.

```
(* C의 struct Token { int type; char* text; } tokens[8192]; 대신 *)
정수[8192] 노드종류풀.
문자열[8192] 노드이름풀.
정수[8192] 노드왼쪽풀.
정수[8192] 노드오른쪽풀.
```

## 해결된 주요 이슈

| 이슈 | 증상 | 원인 | 해결 |
|------|------|------|------|
| 헥스이스케이프 | "포함" 키워드 매칭 실패 | 부트스트랩 코드생성기가 `\xED` → `\\xED` 이중 이스케이프 | 실제 한글 "포함"으로 교체 |
| 포함 오탐 | STATUS_STACK_BUFFER_OVERRUN | 소스 중간에서 "포함" 매칭, 잘못된 파일명 추출 | 줄 시작에서만 매칭 + 길이 제한 |
| argv 인코딩 | 한글 파일경로 깨짐 | Windows argv는 CP949, 소스는 UTF-8 | `_ansi_to_utf8` 런타임 헬퍼 |
| main(void) | argc/argv 미선언 오류 | 매개변수 없는 main에서 argv 변환 코드 출력 | 매개변수 존재 여부 조건 확인 |

| 나머지 연산자 | `%` 구문 오류 | `%%`가 printf %s로 `%%` 리터럴 출력 | `"%"` 단일 퍼센트로 수정 |

## 알려진 제약사항

### 형태소 분리 주의

특정 한글 단어는 접사와 충돌하여 의도치 않게 분리될 수 있다.
분리제외 사전에 등록하여 방지한다.

| 단어 | 잘못된 분리 | 해결 |
|------|-----------|------|
| 경로 | 경 + 로 | 분리제외 등록 |
| 커서 | 커 + 서 | 분리제외 등록 |
| 남은 | 남 + 은 | `남은수`로 변경 |
| 거짓 | 예약어 | `거짓분기`로 변경 |

### 예약어 충돌

변수명으로 사용할 수 없는 예약어:
`참`, `거짓`, `없음`, `정수`, `실수`, `문자`, `문자열`, `공허`, `참거짓`,
`반환`, `반복`, `탈출`, `계속`, `묶음`, `나열`, `합침`, `별칭`, `외부`, `정적`, `상수`

### SOV 체인 호출 제한

SOV 체인 호출(`A를 B다를 C다`)은 불안정할 수 있다.
임시변수를 사용하는 것을 권장한다.

```
(* 불안정 *)
소스를 strlen다를 printf다.

(* 안정 *)
정수 길이는 소스를 strlen다.
길이를 printf다.
```
