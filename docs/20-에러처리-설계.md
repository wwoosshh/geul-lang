# 20. 에러 처리 설계서

## 1. 현황 분석

### 1.1 현재 에러 처리 방식

글 언어는 현재 C 스타일 에러 처리를 그대로 사용한다:

```글
공허 참조 파일은 "data.txt"를 "r"로 파일_열기하다.
파일 == 0이면 {
    "파일을 열 수 없습니다\n"을 쓰기하다.
    1을 종료하다.
}
```

이 패턴은 다음 한계가 있다:

| 문제 | 설명 |
|------|------|
| 매번 수동 검사 | 모든 호출 뒤에 `== 0이면` 패턴 반복 |
| 에러 정보 부재 | 실패 원인(권한, 존재여부, 디스크풀)을 구분할 수 없음 |
| 전파 불가 | 호출 스택을 따라 에러를 올려보내는 문법이 없음 |
| 자원 누수 위험 | 중간에 실패하면 이미 열린 자원을 닫는 코드가 복잡해짐 |

### 1.2 C 코드 생성과의 호환성

글 컴파일러는 `.글 → C → 네이티브` 파이프라인을 사용하므로, 에러 처리 문법은 궁극적으로 C 코드로 변환 가능해야 한다. C에서 사용 가능한 메커니즘:

- **반환값 검사** (가장 자연스러운 매핑)
- **setjmp/longjmp** (비지역 점프 — 복잡하지만 가능)
- **goto cleanup** (자원 정리 패턴)

## 2. 구문 옵션 평가

### 옵션 A: 시도/실패 (구조적 예외 처리)

```글
시도 {
    공허 참조 파일은 "data.txt"를 "r"로 파일_열기하다.
    (* 파일 사용 *)
} 실패 {
    "파일을 열 수 없습니다\n"을 쓰기하다.
}
```

**평가:**

| 기준 | 점수 | 근거 |
|------|------|------|
| 한국어 자연스러움 | 6/10 | "시도하다/실패하다"는 한국어 단어이나 블록 구조가 영어 try/catch 번역투 |
| 구현 난이도 | 상 | setjmp/longjmp 기반 필요, 스택 되감기 관리 복잡 |
| 기존 코드 호환 | 하 | 기존 반환값 패턴과 완전히 다른 메커니즘 |
| C 코드 생성 | 중 | setjmp/longjmp로 변환 가능하나 코드 비대화 |

**장점:** 에러 전파가 자동, 깊은 호출 스택에서 편리

**단점:** C 백엔드에서 setjmp/longjmp는 성능 저하, 자원 정리 시점 불명확, 한국어답지 않은 블록 구조

### 옵션 B: 결과 타입 (다중 반환 — Go 스타일)

```글
[문자열 경로를 안전하게_열기]는 → 공허 참조, 정수 {
    공허 참조 파일은 경로를 "r"로 파일_열기하다.
    파일 == 0이면 { 반환 0, -1. }
    반환 파일, 0.
}

파일, 오류는 "data.txt"를 안전하게_열기하다.
오류 != 0이면 { "실패\n"을 쓰기하다. }
```

**평가:**

| 기준 | 점수 | 근거 |
|------|------|------|
| 한국어 자연스러움 | 5/10 | 다중 반환은 한국어 문법과 무관한 개념 |
| 구현 난이도 | 상 | 다중 반환값 파싱, 타입 시스템 확장, 구조체 자동 래핑 필요 |
| 기존 코드 호환 | 하 | 기존 단일 반환 함수 시그니처 전부 변경 필요 |
| C 코드 생성 | 중 | 구조체로 래핑하여 반환 가능하나 호출 규약 복잡 |

**장점:** 에러를 값으로 다루므로 명시적, Go 언어에서 검증된 패턴

**단점:** 다중 반환 구문이 현재 파서에 없음, 호출부마다 `오류 != 0이면` 보일러플레이트, 한국어 자연스러움 낮음

### 옵션 C: 검사 패턴 (조건부 처리 강화)

```글
공허 참조 파일은 "data.txt"를 "r"로 파일_열기하다.
파일이 없으면 {
    "파일을 열 수 없습니다\n"을 쓰기하다.
    1을 종료하다.
}
```

더 복잡한 예시:

```글
(* 함수 내부에서 에러 전파 *)
[문자열 경로에서 읽기]는 → 문자열 {
    공허 참조 파일은 경로를 "r"로 파일_열기하다.
    파일이 없으면 { 반환 0. }

    문자열 내용은 파일에서 1024를 파일_읽기하다.
    내용이 없으면 {
        파일을 파일_닫기하다.
        반환 0.
    }

    파일을 파일_닫기하다.
    반환 내용.
}

(* 호출부 *)
문자열 데이터는 "config.txt"에서 읽기하다.
데이터가 없으면 {
    "설정 파일을 읽을 수 없습니다\n"을 쓰기하다.
    1을 종료하다.
}
```

**평가:**

| 기준 | 점수 | 근거 |
|------|------|------|
| 한국어 자연스러움 | 9/10 | "~이 없으면"은 완전한 한국어 문장, 기존 `-면` 조건문의 자연 확장 |
| 구현 난이도 | 하 | `없으면` 키워드 추가 + null 검사 코드 생성만 필요 |
| 기존 코드 호환 | 상 | 기존 `== 0이면` 패턴을 문법 설탕으로 대체, 호환 100% |
| C 코드 생성 | 하 | `if (x == NULL)` 또는 `if (x == 0)` 그대로 매핑 |

**장점:** 최소 변경으로 즉시 구현 가능, 기존 코드와 완전 호환, 한국어로 읽히는 에러 처리

**단점:** 구조적 예외 처리가 아님, 에러 전파는 수동

## 3. 권장안: 옵션 C 확장 — "검사 패턴 + 정리 블록"

옵션 C를 기반으로, 자원 정리와 에러 코드를 체계화하는 확장을 추가한다.

### 3.1 핵심 키워드

| 키워드 | 의미 | C 변환 |
|--------|------|--------|
| `없으면` | null/0 검사 | `if (x == NULL)` |
| `실패면` | 음수 반환값 검사 | `if (x < 0)` |
| `정리` | 함수 끝 정리 블록 (goto cleanup) | `cleanup:` 레이블 |
| `넘기다` | 정리 블록으로 이동 | `goto cleanup;` |

### 3.2 문법 정의

#### 3.2.1 `없으면` — null 검사 조건문

```글
(* 기본 사용 *)
파일이 없으면 {
    "에러\n"을 쓰기하다.
}

(* 변수 선언과 동시 사용 *)
공허 참조 파일은 "data.txt"를 "r"로 파일_열기하다.
파일이 없으면 {
    1을 종료하다.
}
```

**C 변환:**
```c
FILE *파일 = fopen("data.txt", "r");
if (파일 == NULL) {
    exit(1);
}
```

#### 3.2.2 `실패면` — 음수/에러 반환값 검사

```글
정수 결과는 소켓에 데이터를 보내기하다.
결과가 실패면 {
    "전송 실패\n"을 쓰기하다.
}
```

**C 변환:**
```c
int 결과 = 보내기(소켓, 데이터);
if (결과 < 0) {
    printf("전송 실패\n");
}
```

#### 3.2.3 `정리` + `넘기다` — 자원 정리 패턴

```글
[문자열 경로를 안전하게_처리]는 → 정수 {
    정수 결과는 -1.
    공허 참조 파일은 0.
    문자열 버퍼는 0.

    파일 = 경로를 "r"로 파일_열기하다.
    파일이 없으면 { 넘기다. }

    버퍼 = 1024를 할당하다.
    버퍼가 없으면 { 넘기다. }

    (* 정상 처리 *)
    결과 = 0.

정리 {
    버퍼가 없지않으면 { 버퍼를 해제하다. }
    파일이 없지않으면 { 파일을 파일_닫기하다. }
    반환 결과.
}
}
```

**C 변환:**
```c
int 안전하게_처리(const char *경로) {
    int 결과 = -1;
    FILE *파일 = NULL;
    char *버퍼 = NULL;

    파일 = fopen(경로, "r");
    if (파일 == NULL) { goto cleanup; }

    버퍼 = malloc(1024);
    if (버퍼 == NULL) { goto cleanup; }

    /* 정상 처리 */
    결과 = 0;

cleanup:
    if (버퍼 != NULL) { free(버퍼); }
    if (파일 != NULL) { fclose(파일); }
    return 결과;
}
```

### 3.3 에러 코드 규약

표준 에러 코드를 `표준/에러.글`로 제공한다:

```글
(* 표준/에러.글 — 에러 코드 정의 *)

정의 에러_없음      0.
정의 에러_일반     -1.
정의 에러_메모리   -2.
정의 에러_파일     -3.
정의 에러_인자     -4.
정의 에러_범위     -5.
정의 에러_영나눗셈 -6.
정의 에러_권한     -7.
```

사용 예:

```글
포함 "표준/에러"

[정수 나눌수를 정수 나누는수로 안전나눗셈]은 → 정수, 정수 {
    나누는수 == 0이면 {
        반환 0, 에러_영나눗셈.
    }
    반환 나눌수 / 나누는수, 에러_없음.
}
```

> **주의**: 위 다중 반환 예시는 장기 목표이다. 현재 구현에서는 출력 매개변수 패턴을 사용한다:

```글
[정수 나눌수를 정수 나누는수로 정수 참조 결과에 안전나눗셈]은 → 정수 {
    나누는수 == 0이면 {
        반환 에러_영나눗셈.
    }
    결과에 나눌수 / 나누는수를 쓰다.
    반환 에러_없음.
}
```

## 4. 실용 예제

### 4.1 파일 I/O 에러 처리

```글
포함 "표준/입출력"
포함 "표준/에러"

[문자열 경로를 설정읽기]는 → 정수 {
    공허 참조 파일은 0.
    문자[1024] 버퍼.

    파일 = 경로를 "r"로 파일_열기하다.
    파일이 없으면 {
        "설정 파일을 찾을 수 없습니다: "를 쓰기하다.
        경로를 쓰기하다.
        "\n"을 쓰기하다.
        반환 에러_파일.
    }

    (* 파일 읽기 *)
    정수 읽은수는 버퍼에 1을 1023을 파일에서 파일_읽기하다.
    읽은수 == 0이면 {
        파일을 파일_닫기하다.
        반환 에러_파일.
    }

    버퍼[읽은수] = 0.
    버퍼를 쓰기하다.
    파일을 파일_닫기하다.
    반환 에러_없음.
}
```

### 4.2 메모리 할당 실패

```글
[정수 크기만큼 안전할당]은 → 공허 참조 {
    공허 참조 메모리는 크기만큼 할당하다.
    메모리가 없으면 {
        "메모리 할당 실패\n"을 쓰기하다.
        1을 종료하다.
    }
    반환 메모리.
}
```

### 4.3 정리 블록을 사용한 복합 자원 관리

```글
[문자열 입력경로를 문자열 출력경로로 파일변환]은 → 정수 {
    정수 결과는 에러_일반.
    공허 참조 입력은 0.
    공허 참조 출력은 0.
    문자열 버퍼는 0.

    입력 = 입력경로를 "rb"로 파일_열기하다.
    입력이 없으면 { 넘기다. }

    출력 = 출력경로를 "wb"로 파일_열기하다.
    출력이 없으면 { 넘기다. }

    버퍼 = 4096을 할당하다.
    버퍼가 없으면 { 넘기다. }

    (* 변환 작업 수행 *)
    (* ... *)

    결과 = 에러_없음.

정리 {
    버퍼 != 0이면 { 버퍼를 해제하다. }
    출력 != 0이면 { 출력을 파일_닫기하다. }
    입력 != 0이면 { 입력을 파일_닫기하다. }
    반환 결과.
}
}
```

### 4.4 0으로 나누기 방어

```글
[정수 가를 정수 나로 안전나누기]는 → 정수 {
    나 == 0이면 {
        "0으로 나눌 수 없습니다\n"을 쓰기하다.
        반환 0.
    }
    반환 가 / 나.
}
```

## 5. 구현 계획

### 5.1 1단계: `없으면` 키워드 (난이도: 하)

**형태소분석기 변경:**

접사 테이블에 `없으면`을 추가한다. 분리 규칙:

```
"파일이 없으면" → "파일" + "이" + "없으면"
```

`없으면`은 새 토큰 종류 `종류_없으면`으로 등록한다.

```c
// 형태소분석기.c — 접사 테이블 추가
{"없으면", 종류_없으면},
```

**구문분석기 변경:**

조건문 파싱에서 `종류_없으면`을 처리한다:

```
기존: "파일이" [주어조사 생략] "0과 같으면" → 비교노드(파일, ==, 0)
추가: "파일이 없으면" → 비교노드(파일, ==, 0)   -- 문법 설탕
```

파서의 `문장파싱()` 함수에서 `종류_없으면` 토큰을 만나면:
1. 직전 식별자(주어)를 좌변으로 취함
2. `노드_이항(같음, 주어, 0리터럴)` AST 노드 생성
3. 이후 블록을 조건문 본문으로 파싱

**코드생성기 변경:** 없음 (기존 조건문 코드 생성 그대로 사용)

**예상 작업량:** 형태소분석기 5줄, 구문분석기 20줄 = 약 25줄

### 5.2 2단계: `실패면` 키워드 (난이도: 하)

`없으면`과 동일한 패턴으로 구현한다.

```
"결과가 실패면" → 비교노드(결과, <, 0)
```

**예상 작업량:** 약 15줄 (1단계와 거의 동일한 구조)

### 5.3 3단계: `정리` + `넘기다` (난이도: 중)

**형태소분석기 변경:**

```c
{"정리", 종류_정리},
{"넘기다", 종류_넘기다},
```

**구문분석기 변경:**

함수 본문 파싱에서 `정리` 블록을 인식한다:

```
함수본문 ::= 문장* [정리블록]
정리블록 ::= "정리" "{" 문장* "}"
```

`넘기다`는 AST에서 `노드_넘기다`로 표현한다.

**코드생성기 변경:**

```c
// "정리" → C의 goto cleanup 패턴
// 함수 내 "정리" 블록이 있으면:
//   - 함수 끝에 "cleanup:" 레이블 생성
//   - 정리 블록 내용을 cleanup 레이블 뒤에 배치
//   - "넘기다" → "goto cleanup;"

// 예시 생성 코드:
// goto cleanup;
// ...
// cleanup:
//   if (버퍼) free(버퍼);
//   return 결과;
```

**제약사항:**
- 함수당 하나의 `정리` 블록만 허용
- `정리` 블록은 반드시 함수 본문 마지막에 위치
- `넘기다`는 `정리` 블록이 있는 함수에서만 사용 가능

**예상 작업량:** 형태소분석기 5줄, 구문분석기 40줄, 코드생성기 30줄 = 약 75줄

### 5.4 4단계: `표준/에러.글` (난이도: 하)

에러 코드 정의와 유틸리티 함수를 포함하는 표준 라이브러리 모듈:

```글
(* 표준/에러.글 *)

정의 에러_없음      0.
정의 에러_일반     -1.
정의 에러_메모리   -2.
정의 에러_파일     -3.
정의 에러_인자     -4.
정의 에러_범위     -5.
정의 에러_영나눗셈 -6.
정의 에러_권한     -7.

(* 에러 코드 → 설명 문자열 *)
[정수 코드를 에러설명]은 → 문자열 {
    코드 == 에러_없음이면      { 반환 "성공". }
    코드 == 에러_일반이면      { 반환 "알 수 없는 오류". }
    코드 == 에러_메모리이면    { 반환 "메모리 할당 실패". }
    코드 == 에러_파일이면      { 반환 "파일 접근 실패". }
    코드 == 에러_인자이면      { 반환 "잘못된 인자". }
    코드 == 에러_범위이면      { 반환 "범위 초과". }
    코드 == 에러_영나눗셈이면  { 반환 "0으로 나누기". }
    코드 == 에러_권한이면      { 반환 "권한 부족". }
    반환 "정의되지 않은 오류".
}
```

**예상 작업량:** 약 30줄

## 7. 장기 확장 방향

### 7.1 `없으면` 체이닝

```글
(* 미래: 대안값 지정 *)
문자열 이름은 설정에서_읽기하다 없으면 "기본값".
```

**C 변환:** `char *이름 = 설정에서_읽기(); if (이름 == NULL) 이름 = "기본값";`

### 7.2 에러 전파 축약 (`올리다`)

```글
(* 미래: 에러 시 즉시 반환 *)
공허 참조 파일은 경로를 "r"로 파일_열기하다 없으면 올리다.
(* → if (파일 == NULL) return NULL; 또는 return 에러코드; *)
```

이는 Rust의 `?` 연산자에 해당하지만, "에러를 호출자에게 올리다"라는 한국어 표현을 사용한다.

### 7.3 다중 반환값 (장기)

현재 글 언어는 단일 반환값만 지원한다. 향후 다중 반환이 구현되면 Go 스타일 에러 패턴이 가능해진다:

```글
(* 미래: 다중 반환 *)
파일, 오류는 경로를 안전하게_열기하다.
오류가 실패면 { ... }
```

이는 파서에 다중 반환 타입(`→ 타입1, 타입2`)과 다중 대입 구문이 필요하므로, 별도 설계서에서 다룬다.

## 8. 설계 원칙 요약

1. **한국어 우선**: "없으면", "실패면", "정리", "넘기다" — 모두 한국어 어휘
2. **점진적 도입**: 기존 `== 0이면` 코드가 계속 작동, 새 키워드는 문법 설탕
3. **C 호환**: 모든 구문이 C로 직접 변환 가능 (setjmp 불필요)
4. **최소 변경**: 1~2단계는 합계 40줄 변경으로 즉시 효과
5. **실용성**: 글 컴파일러 자체 코드에서 바로 사용 가능한 패턴
