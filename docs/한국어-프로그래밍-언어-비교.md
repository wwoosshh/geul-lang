│  └───────────┘  └───────────┘                       │
├─────────────────────────────────────────────────────┤
│  범용/교육 언어 (인터프리터/VM)                        │

## 4. 글 언어의 영역 — 프로그래밍 언어 분류

프로그래밍 언어는 용도와 특성에 따라 분류됩니다:

### 언어 유형별 분류

| 유형 | 특징 | 대표 언어 | 글의 위치 |
|------|------|-----------|-----------|
| **시스템 언어** | 네이티브 코드, 메모리 직접 관리 | C, Rust, Zig, Go | **✅ 해당** |
| 응용 언어 | VM/런타임, GC, 풍부한 생태계 | Java, C#, Kotlin | — |
| 스크립트 언어 | 인터프리터, 빠른 프로토타이핑 | Python, JS, Ruby | — |
| 함수형 언어 | 불변성, 고차함수, 순수 함수 | Haskell, OCaml | — |
| 교육 언어 | 단순 문법, 초보자 친화 | Scratch, Logo | 일부 해당 |

### 글의 영역: **시스템 언어**

글은 다음 특성으로 **시스템 언어**로 분류됩니다:

1. **네이티브 기계어 직접 생성** — x86-64 PE 바이너리
2. **수동 메모리 관리** — `할당`/`해제` (malloc/free)
3. **포인터 연산** — 직접 주소 접근 가능
4. **C ABI 호환** — C 표준 라이브러리 직접 호출
5. **구조체/공용체/열거형** — 저수준 데이터 레이아웃 제어
6. **자체호스팅 컴파일러** — 스스로를 컴파일할 만큼 언어가 완전함

### C/Go/Zig와 같은 계층, 다른 철학

```
C     — 영어 문법, 최소주의, 1972년
Go    — 영어 문법, 단순+실용, 2009년
Zig   — 영어 문법, 안전+성능, 2016년
글    — 한국어 문법, 교착어 설계, 2026년
```

같은 "외부 도구 없이 네이티브 바이너리를 생성하는 시스템 언어"이지만, 글은 **비영어 자연어 문법을 채용한 유일한 시스템 언어**입니다.

## 5. 다른 한국어 언어와의 결정적 차이

### vs 한 (Han) — 가장 가까운 경쟁자

| | 한 | 글 |
|---|---|---|
| 문법 | `만약 n <= 1 이면 { 반환 n }` | `n <= 1이면 { 반환 n. }` |
| 설계 | C/Rust 키워드를 한국어로 번역 | 한국어 교착어 문법 자체를 활용 |
| 함수 호출 | `피보나치(n - 1)` (C 스타일) | `(n - 1)을 피보나치다` (SOV) |
| 백엔드 | LLVM (외부 의존) | PE 직접 생성 (독립) |
| 독립성 | Rust + LLVM + clang 필요 | **컴파일러 파일 1개** |
| 자체호스팅 | ❌ | **✅** |

한(Han)은 기능적으로 더 풍부하지만 (REPL, LSP, Python 연동, 패턴매칭), 글(Geul)은 **언어학적 설계**와 **기술적 독립성**에서 차별됩니다.

### vs 누리 (Nuri) — 문법적으로 가장 유사

| | 누리 | 글 |
|---|---|---|
| 문법 | `[ㄱ]을 [ㄴ]으로 나누다` | `왼쪽을 오른쪽에 더하다` |
| 패러다임 | 순수 함수형 (변수 없음) | 명령형 + 구조체 + 포인터 |
| 실행 | Haskell 인터프리터 | 네이티브 코드 직접 생성 |
| 실용성 | 학술/교육 | 시스템 프로그래밍 가능 |

누리는 조사 활용이 글과 유사하지만 순수 함수형 인터프리터로, 실용적 프로그래밍에는 제약이 있습니다.

### vs 아희 (Aheui) — 유일한 다른 자체호스팅

아희는 한글 자모를 명령어로 사용하는 난해한(esolang) 언어입니다. 자체호스팅을 달성했지만 실용적 프로그래밍 언어가 아닙니다. 글은 **실용 언어로서 자체호스팅을 달성한 유일한 한국어 언어**입니다.

## 7. 요약

### 글 언어만의 고유한 조합

1. **한국어 교착어 문법** (조사·어미 기반) — 키워드 번역이 아닌 문법 구조
2. **네이티브 코드 직접 생성** — LLVM/GCC/VM 없이 PE 바이너리
3. **자체호스팅** — 컴파일러가 자기 자신을 컴파일 (고정점 증명)
4. **단일 파일 배포** — 컴파일러 1개 + 표준라이브러리 1개

이 4가지를 동시에 갖춘 한국어 프로그래밍 언어는 **글이 유일**합니다.
