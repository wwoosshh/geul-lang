## 3. 한글 언어적 특성과 프로그래밍 매핑

### 3.1 교착어 특성 → 연결형(Concatenative) 프로그래밍
- 어근 + 접미사 체인 = 함수 합성
- 기존 어떤 언어도 형태소 접착을 합성 메커니즘으로 사용하지 않음
- Forth, Factor, Joy와 구조적으로 동일하나 공백 대신 접착 사용

### 3.2 조사(助詞) → 역할 기반 인자 바인딩
- 을/를 = 목적어(피연산자), 이/가 = 주어(호출자), 은/는 = 주제(컨텍스트)
- 에 = 목적지, 로 = 수단, 에서 = 출처, 까지 = 종점, 부터 = 시작점
- 나데시코가 이미 증명: 조사로 인자 순서 자유화 가능

### 3.3 어미 변화 → 제어 흐름
- -다(서술) = 문장종결, -면(조건) = if, -고(접속) = 순차실행
- -어서(인과) = 파이프라인, -거나(선택) = or, -는동안(지속) = while
- -라(명령) = 실행, -까(의문) = 테스트/assert

### 3.4 주어 생략(Pro-drop) → 암시적 컨텍스트
- Perl의 $_, Kotlin의 it, 스택 기반 언어의 스택 최상단과 동일
- 가장 최근 언급된 대상이 암시적 인자

### 3.5 주제-평언 구조 → 스코프/컨텍스트
- 은/는으로 주제 설정 → 이후 문장이 해당 주제에 대해 작동
- Python의 with, 블록 스코프와 유사하나 더 자연스러움

## 4. 기술적 실현 가능성 근거

1. 나데시코 증명: 일본어(한글과 거의 동일 문법)로 조사 기반 프로그래밍 구현, 교과서 채택
2. 연결형 언어 이론 성숙: 정적 타입 체크 가능한 수준(Kitten)
3. 형태소 분석 도구: MeCab-ko, KoNLPy 등 존재
4. 한글 유니코드 규칙성: 음절 = 0xAC00 + (초성×21 + 중성)×28 + 종성
5. Inform 7 교훈: 정확한 부분집합 정의시 파서 구현 가능, 교착어가 오히려 유리
