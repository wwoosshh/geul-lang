# 19. 문자열 강화 설계

## 개요

글 언어의 문자열 기능을 강화한다. 현재 `문자열` 타입은 C의 `char*`에 직접 대응하며 안전하지 않다. 문자열 보간, 길이 조회, 안전한 연결 기능을 추가한다.

## 기능 1: 문자열 보간 (String Interpolation)

### 문법

```글
문자열 이름은 "철수".
정수 나이는 25.

(* 중괄호 보간 *)
문자열 인사는 "이름: {이름}, 나이: {나이}세".

(* 표현식 보간 *)
문자열 결과는 "합계: {가 + 나}".

(* 중괄호 이스케이프 *)
문자열 예시는 "중괄호는 \{이렇게\} 쓴다".
```

### 구현 전략

#### 렉서 변경
- 문자열 리터럴 파싱 시 `{` 감지
- 보간 포함 문자열은 새 토큰 종류 `종류_보간문자열`로 분류
- 보간 부분을 별도 토큰으로 분리:
  ```
  "이름: {이름}, 나이: {나이}세"
  →  ["이름: ", 보간시작, 이름, 보간끝, ", 나이: ", 보간시작, 나이, 보간끝, "세"]
  ```

#### 파서 변경
- `노드_보간문자열` AST 노드 추가
- 정적 부분(문자열)과 동적 부분(표현식)의 교대 목록 저장

#### C 코드 생성

```글
문자열 인사는 "이름: {이름}, 나이: {나이}세".
```

생성 코드:

```c
char 인사[256];
snprintf(인사, 256, "이름: %s, 나이: %d세", 이름, 나이);
```

타입별 형식 지정자:
- `정수` → `%d`
- `긴정수` → `%lld`
- `실수` → `%f`
- `문자열` → `%s`
- `문자` → `%c`
- 기타(포인터) → `%p`

### 고려사항
- 버퍼 크기: 정적 분석이 어려우므로 기본 256바이트, `malloc`+`snprintf` 조합 가능
- 중첩 보간: `"{a의 {b}}"` 같은 중첩은 1단계에서 미지원
- UTF-8 안전성: `snprintf`는 바이트 단위이므로 한글 잘림 주의

## 기능 2: 문자열 길이

### 문법

```글
문자열 텍스트는 "안녕하세요".

(* 방법 1: 함수 호출 *)
정수 길이는 글자수(텍스트).

(* 방법 2: 멤버 접근 스타일 — 향후 확장 *)
정수 길이는 텍스트의 길이.

(* 바이트 수 *)
정수 바이트는 바이트수(텍스트).
```

### 구현

#### 글자수 (UTF-8 문자 수)

```c
int 글자수(const char *s) {
    int count = 0;
    while (*s) {
        if ((*s & 0xC0) != 0x80) count++;
        s++;
    }
    return count;
}
```

#### 바이트수

```c
int 바이트수(const char *s) {
    return (int)strlen(s);
}
```

이미 `표준/문자열.글`에 `글자수` 함수가 존재할 수 있음. 기존 구현과 통합 필요.

## 기능 3: 안전한 문자열 연결

### 문법

```글
문자열 이름은 "김".
문자열 성은 "철수".

(* + 연산자로 연결 *)
문자열 전체는 이름 + 성.

(* 여러 개 연결 *)
문자열 인사는 "안녕, " + 이름 + 성 + "!".
```

### 구현 전략

#### 방법 A: 정적 버퍼 (단순)

`+` 연산자의 양쪽이 모두 `문자열` 타입이면 연결로 처리:

```c
/* "안녕" + "세계" */
char __tmp_1[512];
snprintf(__tmp_1, 512, "%s%s", "안녕", "세계");
```

단점: 버퍼 크기 제한, 임시 변수 관리 복잡

#### 방법 B: malloc 기반 (안전)

```c
char *__글_연결(const char *a, const char *b) {
    size_t la = strlen(a), lb = strlen(b);
    char *r = malloc(la + lb + 1);
    memcpy(r, a, la);
    memcpy(r + la, b, lb + 1);
    return r;
}
```

단점: 메모리 해제 필요 (누수 위험)

#### 방법 C: 표현식 트리 플래트닝 (권장)

`a + b + c`를 단일 `snprintf` 호출로 변환:

```c
char __tmp[512];
snprintf(__tmp, 512, "%s%s%s", a, b, c);
```

컴파일러가 `+` 체인을 인식하여 하나의 `snprintf`로 병합.

### 파서/코드생성기 변경

- 이항연산 `+`의 양쪽 타입이 `문자열`이면 `노드_문자열연결`로 변환
- 타입 추론 필요: 리터럴은 즉시 판별, 변수는 심볼 테이블에 타입 정보 필요

## 1단계 구현 완료 (2026-03-21): 표준 라이브러리 함수

컴파일러 수정 없이 `표준/문자열.글`에 문자열 강화 함수를 추가하였다.

### 구현된 함수

| 함수 | 설명 | 호출 방식 |
|------|------|-----------|
| `한글_글자수(문자열)` | UTF-8 글자 수 (한글=1) | 직접호출 |
| `바이트수(문자열)` | 바이트 길이 (`strlen`) | SOV 가능 |
| `문자열_합치기(a, b)` | `malloc` 기반 안전 연결, 해제 필요 | 직접호출 |
| `문자열_포함(건초, 바늘)` | 부분문자열 포함 여부 (1/0) | 직접호출 |
| `문자열_자르기(원본, 시작, 길이)` | 바이트 기준 추출, `malloc` 반환, 해제 필요 | 직접호출 |

### 기존 함수 (이미 구현됨)

| 함수 | 설명 |
|------|------|
| `글자수(문자열)` | 바이트 길이 (`strlen`) — 주의: UTF-8 글자 수 아님 |
| `문자열_비교(a, b)` | `strcmp` |
| `문자열_같은가(a, b)` | `strcmp == 0` |
| `문자열_복사(대상, 원본)` | `strcpy` |
| `문자열_연결(대상, 원본)` | `strcat` (대상 버퍼에 직접 연결) |
| `문자열_찾기(건초, 바늘)` | `strstr` |
| `문자열_복제(원본)` | `strdup` |

### 주의사항

- `글자수`는 `strlen` (바이트 수)이고, `한글_글자수`가 UTF-8 문자 수이다.
- `문자열_합치기`와 `문자열_자르기`는 `malloc`으로 새 메모리를 할당하므로 사용 후 `해제` 필요.
- `문자열_자르기`의 `시작`과 `길이`는 바이트 기준이므로 한글 문자열 자르기 시 UTF-8 바이트 경계에 주의.
