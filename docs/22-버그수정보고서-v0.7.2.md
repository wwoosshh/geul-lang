## 버그 #1: `&배열[i]` 쓰레기 주소 → 역참조 크래시

### 증상

```글
정수[4] 배열.
배열[0] = 10.
정수 참조 포인터 = &배열[0].
쓰기("주소: %p\n", 포인터).   (* 출력: 0x0000000000000002 (쓰레기) *)
*포인터                        (* 세그폴트 *)
```

단순 `&스칼라값`은 정상이지만 `&배열[인덱스]`로 얻은 포인터는 잘못된 주소를 반환하여 역참조 시 세그폴트가 발생했습니다.

### 루트 원인 — 이중 문제

**문제 A (파서 우선순위)**: `src/parser.gl:549`
```글
(* 주소참조: &식별자 *)
((현재종류 == 종류_비트와))이면 {
    전진().
    정수 대상 = 기본표현식().   (* ← postfix 제외! *)
    ...
}
```

`&` 단항이 `기본표현식()`(후위 연산 없음)만 호출하여 `&배열[0]`이 `(&배열)[0]`로 파싱됐습니다. C 문법과 달리 postfix `[]`가 `&`보다 낮은 우선순위로 적용.

증거 (IR 덤프):
```
t6 = 주소 t2                   ← &배열 먼저
t7 = 상수정수 0
t8 = 원소적재 t6, t7 [크기=1]  ← 그 다음 [0] byte-access
```

**문제 B (IR 변환)**: `src/ir_transform.gl:857`
```글
(종류 == 노드_주소참조)이면 {
    정수 대상 = 노드왼쪽풀[노드]를 표현식변환다.  (* 자식이 배열접근이면 값 로드 *)
    명령등록(명령_주소, 결과, 대상, ...).        (* 값 슬롯의 주소를 취득 *)
    반환 결과.
}
```

설령 파서가 올바르게 `&(배열[0])`을 만들어도, 자식 `표현식변환`이 `명령_원소적재`를 발행하여 배열 원소 **값**을 임시슬롯에 로드한 뒤, `명령_주소`가 **그 임시슬롯의 스택 주소**를 반환하는 이중 오류.

### 수정

**파서 수정** (`parser.gl`):
```글
(* 주소참조: &식별자, &배열[인덱스], &묶음.필드 *)
((현재종류 == 종류_비트와))이면 {
    전진().
    정수 대상 = 접미표현식().   (* ← postfix 포함 *)
    정수 노드 = 노드생성(노드_주소참조, 줄, 열).
    노드왼쪽풀[노드] = 대상.
    반환 노드.
}
```

**IR 신규 opcode `명령_원소주소` 추가** (`ir.gl`):
```글
명령_원소적재,
명령_원소저장,
명령_원소주소,   (* ← 신규: 값 로드 없이 주소만 계산 *)
```

**IR 변환 분기** (`ir_transform.gl`):
```글
(종류 == 노드_주소참조)이면 {
    정수 자식노드 = 노드왼쪽풀[노드].
    (* &배열[인덱스] → 원소 주소 계산 (값 로드 금지) *)
    (노드종류풀[자식노드] == 노드_배열접근)이면 {
        정수 배열기준 = 노드왼쪽풀[자식노드]를 표현식변환다.
        정수 인덱스기준 = 노드오른쪽풀[자식노드]를 표현식변환다.
        정수 결과주소 = -1을 새임시변수다.
        정수 명령번호주소 = 명령등록(명령_원소주소, 결과주소, 배열기준, ...).
        명령번호주소 >= 0이면 {
            (배열기준 >= 0) * (임시변수원소크기풀[배열기준] == 8) > 0이면 {
                명령정수값풀[명령번호주소] = 8.
            }
            아니면 { 명령정수값풀[명령번호주소] = 1. }
        }
        반환 결과주소.
    }
    (* 기존 경로: 식별자/변수 주소 *)
    ...
}
```

**PE 인코더** (`pe_gen.gl`):
```글
(* 원소 주소: 결과 = &배열[인덱스] — base + index*size, deref 없음 *)
(종류 == 명령_원소주소)이면 {
    PE_적재_r10(인자1).              (* 배열 base *)
    PE_적재_r11(인자2).              (* 인덱스 *)
    PE_캐시무효화().
    정수 원소크기3 = 명령정수값풀[명령인덱스].
    원소크기3 == 8이면 { 인코딩_imul_r11_imm(8). }
    원소크기3 == 4이면 { 인코딩_imul_r11_imm(4). }
    인코딩_add_r10_r11().             (* r10 = base + index*size *)
    PE_저장_r10(결과).                (* deref 없이 주소 그대로 저장 *)
    반환 1.
}
```

## 버그 #2: 포인터 산술 스케일링 실패 (`*(포인터 + N)` 값 0)

### 증상

```글
정수[4] 배열.
배열[0] = 10.  배열[1] = 20.  배열[2] = 30.
정수 참조 포인터 = &배열[0].
*포인터               (* 10 — 정상 *)
*(포인터 + 1)         (* 0 — 잘못됨, 20이어야 함 *)
*(포인터 + 2)         (* 0 — 잘못됨, 30이어야 함 *)
```

`포인터 + N`이 바이트 단위로만 더해져 `정수[4]` 배열의 다음 원소(+8바이트)가 아닌 중간 바이트(+1바이트)를 읽고 있었습니다.

### 루트 원인

포인터 산술 스케일링 로직(`ir_transform.gl:599`)은 `임시변수원소크기풀`을 참고:
```글
((연산자 == 종류_더하기) + (연산자 == 종류_빼기) > 0) * (왼쪽 >= 0) > 0이면 {
    임시변수원소크기풀[왼쪽] > 1이면 {
        포인터스케일 = 임시변수원소크기풀[왼쪽].
    }
}
```

하지만 **지역 변수 선언 경로에서 `참조타입`에 대해 `임시변수원소크기풀`을 설정하지 않았습니다**. 기존 코드(`ir_transform.gl:1023`)는 `노드_배열타입`만 처리:
```글
(타입노드 >= 0) * (노드종류풀[타입노드] == 노드_배열타입) > 0이면 {
    임시변수크기풀[변수] = 배열총크기계산(타입노드).
    임시변수원소크기풀[변수] = 배열원소크기결정(타입노드).
}
(* ← 참조타입 분기 없음 *)
```

### 초기 시도의 함정

단순히 `참조타입`에도 `임시변수원소크기풀 = 8`을 설정하면 **자체호스팅이 붕괴**합니다. 이유:

`PE_적재_r10`(`pe_gen.gl:1725`)은 `임시변수원소크기풀 > 0`을 **"배열이다"** 신호로 해석하여 `lea`(주소) 경로로 진입:
```글
(임시변수원소크기풀[임시번호] > 0) * (임시변수종류풀[임시번호] != 3) > 0이면 {
    주소적재 = 1.   (* ← lea 경로 *)
}
```

포인터 변수에 이 신호를 세우면 **포인터 값 대신 포인터 변수의 스택 슬롯 주소**가 로드됩니다. `공허 참조 파일 = 파일_열기(...)` 같은 코드가 모두 망가져 컴파일러 자체가 빌드 실패.

즉 기존 필드가 **"배열이다"와 "포인터 원소크기"** 두 신호를 겸용하고 있어 분리가 필요했습니다.

### 수정

**전용 필드 신설** (`ir.gl`):
```글
정수[65536] 임시변수원소크기풀.         (* 기존: 배열 원소 크기 *)
정수[65536] 임시변수포인터원소크기풀.   (* 신규: 포인터 산술 스케일링 전용 *)
```

**참조 대상 타입별 크기 결정 헬퍼** (`ir_transform.gl`):
```글
[정수 참조타입노드를 참조원소크기결정]은 → 정수 {
    참조타입노드 < 0이면 { 반환 0. }
    정수 대상타입 = 노드왼쪽풀[참조타입노드].
    대상타입 < 0이면 { 반환 0. }
    정수 대상종류 = 노드종류풀[대상타입].
    대상종류 == 노드_기본타입이면 {
        정수 원시타입 = 노드연산자풀[대상타입].
        (원시타입 == 종류_문자) * (원시타입 != 종류_문자열) > 0이면 { 반환 1. }
        (원시타입 == 종류_공허)이면 { 반환 1. }
        반환 8.
    }
    반환 8.
}
```

**변수 선언에서 설정** (`ir_transform.gl`):
```글
(* 참조(포인터) 타입이면 포인터 산술 스케일링 크기 저장 *)
(타입노드 >= 0) * (노드종류풀[타입노드] == 노드_참조타입) > 0이면 {
    임시변수포인터원소크기풀[변수] = 참조원소크기결정(타입노드).
}
```

**스케일링 로직 확장** (`ir_transform.gl`):
```글
((연산자 == 종류_더하기) + (연산자 == 종류_빼기) > 0) * (왼쪽 >= 0) > 0이면 {
    임시변수원소크기풀[왼쪽] > 1이면 {
        포인터스케일 = 임시변수원소크기풀[왼쪽].
    }
    아니면 임시변수포인터원소크기풀[왼쪽] > 1이면 {   (* ← 추가 *)
        포인터스케일 = 임시변수포인터원소크기풀[왼쪽].
    }
}
```

### 검증

```
기본: 10
오프셋1: 20    (* ← 수정 전: 0 *)
오프셋2: 30    (* ← 수정 전: 0 *)
```

## 버그 #3: 묶음 포인터 매개변수 `의` 접근 쓰레기값

### 증상

```글
묶음 점은 정수 가로, 정수 세로.

[점 참조 포인터를 의표시]는 {
    쓰기("의-접근: 가로=%d 세로=%d\n", 포인터의 가로, 포인터의 세로).
}
(* 출력: 의-접근: 가로=306183712 세로=306183776 — 쓰레기값 *)
```

포인터 매개변수에 대해 `→`(화살표)는 정상이지만 `의`(점 접근)는 스택 주소 같은 쓰레기값을 반환.

### 루트 원인

`pe_gen.gl:3035`의 `명령_필드적재` 처리:
```글
(종류 == 명령_필드적재)이면 {
    정수 화살표여부 = 명령타입풀[명령인덱스].
    화살표여부 == 1이면 {
        PE_적재_r10(인자1).                 (* → : 포인터 값을 r10에 *)
    }
    아니면 (인자1 >= 0) * (임시변수종류풀[인자1] == 2) > 0이면 {
        ... 전역 구조체 lea ...
    }
    아니면 {
        정수 구조체오프셋 = PE_임시오프셋[인자1].
        인코딩_lea_r10_rbp(구조체오프셋).   (* 의 : 지역 구조체로 가정 → lea *)
    }
    ...
    인코딩_mov_r10_deref_r10().             (* 필드 값 로드 *)
}
```

포인터 매개변수를 **지역 구조체로 오해**하여 `lea r10, [rbp+오프셋]`을 실행. 이는 **포인터 변수의 스택 슬롯 주소**(= 포인터의 포인터)를 반환. 필드 오프셋을 더하고 역참조하면 쓰레기값이 나옵니다.

### 수정 (`pe_gen.gl`)

포인터 매개변수를 감지하여 자동으로 화살표 시맨틱 적용:
```글
(종류 == 명령_필드적재)이면 {
    정수 화살표여부 = 명령타입풀[명령인덱스].
    (* '의' 접근이어도 포인터 매개변수면 화살표 시맨틱 적용 *)
    (화살표여부 == 0) * (인자1 >= 0) * (임시변수종류풀[인자1] == 3) * (임시변수원소크기풀[인자1] > 0) > 0이면 {
        화살표여부 = 1.
    }
    화살표여부 == 1이면 {
        PE_적재_r10(인자1).
    }
    ...
}
```

`임시변수종류풀[인자1] == 3`은 매개변수, `임시변수원소크기풀[인자1] > 0`은 포인터형 매개변수(기존 `ir_transform.gl:1367`에서 `참조타입` 매개변수에 8 설정). `명령_필드저장` 쪽도 동일하게 수정.

### 검증

```
의-접근: 가로=10 세로=20     (* ← 수정 전: 306183712 / 306183776 *)
```

## 버그 #4: 전역 변수 음수 초기화가 0으로 평가

### 증상

```글
정수 에러 = 0 - 3.    (* 런타임 값: 0 — 잘못됨 *)
정수 직접음수 = -5.   (* 런타임 값: 0 — 잘못됨 *)
```

전역 변수의 초기화식이 상수 리터럴이 아닌 **표현식**이면 무시되어 BSS 기본값 0이 유지되고 있었습니다.

### 루트 원인

`pe_gen.gl:4325`의 전역 초기화 루프:
```글
정수 초기노드종류 = 노드종류풀[초기노드].
(* 정수 리터럴만 처리 *)
(초기노드종류 == 노드_정수리터럴)이면 {
    정수 초기값 = 노드정수풀[초기노드].
    ... mov [rip+disp32], r10 코드 생성 ...
}
(* ← 단항/이항 연산 분기 없음 → 표현식은 그냥 무시됨 *)
```

`-5`는 `노드_단항연산(빼기, 5)`, `0 - 3`은 `노드_이항연산(빼기, 0, 3)`이라서 둘 다 해당 없음.

### 수정 (`pe_gen.gl`)

컴파일타임 상수 접힘 헬퍼 2개 신설:
```글
[정수 노드를 정수상수표현인가]는 → 정수 {
    노드 < 0이면 { 반환 0. }
    정수 종류 = 노드종류풀[노드].
    (종류 == 노드_정수리터럴)이면 { 반환 1. }
    (종류 == 노드_단항연산)이면 {
        정수 연산자 = 노드연산자풀[노드].
        (연산자 == 종류_빼기) + (연산자 == 종류_더하기) > 0이면 {
            반환 정수상수표현인가(노드왼쪽풀[노드]).
        }
        반환 0.
    }
    (종류 == 노드_이항연산)이면 {
        정수 연산자2 = 노드연산자풀[노드].
        (연산자2 == 종류_더하기) + (연산자2 == 종류_빼기) + (연산자2 == 종류_곱하기) + (연산자2 == 종류_나누기) > 0이면 {
            ... 양쪽 재귀 ...
        }
        반환 0.
    }
    반환 0.
}

[정수 노드를 정수상수평가]는 → 긴정수 {
    (* 리터럴: 그대로 반환 *)
    (* 단항 빼기: -X *)
    (* 이항 +/-/*/: 재귀 계산 *)
    ...
}
```

전역 초기화 루프를 일반화:
```글
(* 상수 표현이면 접어서 방출 — 리터럴 + 단항/이항 산술 *)
정수상수표현인가(초기노드) != 0이면 {
    긴정수 초기값긴 = 정수상수평가(초기노드).
    정수 초기값 = 초기값긴.
    ... 기존 코드 생성 ...
}
```

### 검증

```
에러: -3          (* ← 수정 전: 0 *)
직접음수: -5      (* ← 수정 전: 0 *)
```

## 버그 #5: 숫자 상수 매크로가 런타임에 0

### 증상

```글
정의 원주율 3.14159.
정의 상수정수 42.

쓰기("원주율: %f\n", 원주율).       (* 출력: 0.000000 *)
쓰기("상수정수: %d\n", 상수정수).   (* 출력: 14163998 (쓰레기) *)
```

### 루트 원인

매크로는 `compiler.gl`에 등록되지만 `매크로치환`은 **함수 호출 맹글링에서만 사용**되고 있었습니다 (`pe_gen.gl:280`):
```글
[문자열 이름을 PE_호출맹글링]은 → 문자열 {
    문자열 치환됨은 이름을 매크로치환다.   (* ← 여기만 *)
    ...
}
```

즉 `정의 쓰기 printf` 같은 **함수 별칭**만 작동하고, 값 매크로는 참조되지 않아 일반 식별자로 취급 → 미정의 변수 → 쓰레기/0.

### 수정 (`ir_transform.gl`)

식별자 해석 시(변수/나열/함수 검색 실패 후) 매크로 테이블 조회 및 숫자 리터럴 방출:
```글
(* 숫자 매크로 치환 — 정수/실수 값을 상수로 방출 *)
문자열 매크로값 = 매크로치환(식별이름).
매크로값 != 식별이름이면 {
    정수 첫글자 = 매크로값[0].
    정수 시작위치 = 0.
    첫글자 == 45이면 { 시작위치 = 1. 첫글자 = 매크로값[1]. }  (* 음수 *)
    (첫글자 >= 48) * (첫글자 <= 57) > 0이면 {                  (* 숫자 시작 *)
        정수 소수점개수 = 0.
        정수 위치 = 시작위치.
        정수 유효숫자 = 1.
        (매크로값[위치] != 0) * (유효숫자 == 1) > 0동안 {
            정수 글자 = 매크로값[위치].
            글자 == 46이면 { 소수점개수 = 소수점개수 + 1. }
            아니면 ((글자 < 48) + (글자 > 57) > 0)이면 { 유효숫자 = 0. }
            위치 = 위치 + 1.
        }
        (유효숫자 == 1) * (소수점개수 == 1) > 0이면 {
            (* 실수 매크로 → 명령_상수실수 *)
            ...
        }
        (유효숫자 == 1) * (소수점개수 == 0) > 0이면 {
            (* 정수 매크로 → 명령_상수정수 *)
            정수 매인덱스2 = 명령등록(명령_상수정수, 매결과2, ...).
            명령정수값풀[매인덱스2] = strtoll_10(매크로값).
            ...
        }
    }
}
```

### 검증

```
원주율: 3.141590    (* ← 수정 전: 0.000000 *)
상수정수: 42        (* ← 수정 전: 14163998 *)
```

## 버그 #6: 형태소 분석기가 한글 식별자를 잘못 분리 (`와이` → `와`+`이` 크래시)

### 증상

```글
정수 와이 = 100.   (* ← 컴파일러 세그폴트 *)
```

한글 렉서가 `와이`를 조사 분리 규칙에 따라 `와`(접속 조사) + `이`(주격 조사)로 쪼개면서 파서 크래시.

### 루트 원인

`lexer.gl:654`의 선언 이름 모드 분리 로직:
```글
(어근알려짐 == 0) * 조사인가(선접종류) * (선어근길이 >= 3) > 0이면 {
    참거짓 부분제외 = 0.
    (* 제외사전 (나이, 넓이 등) 검사 *)
    ...
    부분제외 == 0이면 {
        어근알려짐 = 1.
        심볼등록(선어근).    (* ← '와' 를 유효 심볼로 자기-확신 *)
    }
}
```

`와이`를 처리할 때:
1. 접미사 `이`(3 bytes) 매칭 → 어근 `와`(3 bytes) 남음
2. `와`는 키워드/제외사전/기존심볼 모두 아님
3. 어근 길이 ≥ 3 바이트 조건 통과
4. 제외사전의 복합어와도 무관
5. → `와`를 심볼로 자기 등록하고 분리 → 토큰 `와` + `이`
6. 파서가 `정수 와 이 = 100.` 상태로 혼란 → 크래시

근본 원인: **분리 후 어근이 그 자체로 조사/어미(접사 텍스트)인 경우를 검사하지 않음**.

### 수정 (`lexer.gl`)

1음절 어근에 대해 접사 텍스트 일치 검사 추가:
```글
(어근알려짐 == 0) * 조사인가(선접종류) * (선어근길이 >= 3) > 0이면 {
    참거짓 부분제외 = 0.
    (* 어근이 1음절이고 그 자체가 접사 텍스트이면 분리 금지 (예: 와이=와+이) *)
    선어근길이 == 3이면 {
        정수 조사검사번 = 0.
        (조사검사번 < 접사수) * (부분제외 == 0)동안 {
            접사바이트수배열[조사검사번] == 3이면 {
                문자열_일부비교(단어시작, 접사텍스트배열[조사검사번], 3) == 0이면 {
                    부분제외 = 1.
                }
            }
            조사검사번 = 조사검사번 + 1.
        }
    }
    ... 기존 제외사전 검사 ...
}
```

### 검증

```
와이=100 원넓이=314
```

## 버그 #7: 문자열 보간 + 형식 지정자 혼용 시 인자 오류

### 증상

```글
정수 값 = 255.
문자열 이름 = "철수".
쓰기("이름={이름} 값=%d 16진=%x\n", 값, 값).
(* 출력: 이름=철수 값=2026569728 16진=fffffffc — 쓰레기 *)
```

`{보간}` 토큰과 `%d`/`%x` 형식 지정자를 한 문자열에서 섞으면 인자 매핑이 어긋납니다.

### 루트 원인

`ir_transform.gl:824`의 `노드_호출` 처리는 첫 인자가 보간 토큰 포함 문자열이면 `보간호출생성`으로 **전체 호출을 교체**:
```글
(첫인자값노드 >= 0) * (노드종류풀[첫인자값노드] == 노드_문자열리터럴) > 0이면 {
    ...
    보간대상 > 0이면 {
        보간호출생성(문자열내용).   (* 추가 인자를 무시하고 세그먼트별 분할 호출 *)
        반환 -1.
    }
}
```

`보간호출생성`은 문자열을 `{이름}`으로 쪼개 각 세그먼트마다 `printf()` 호출을 개별 발행합니다. 뒷부분 `" 값=%d 16진=%x\n"`이 별도 printf로 방출되는데, 이 printf는 **`%d`/`%x`에 해당하는 인자를 받지 못해** 스택 쓰레기를 읽음.

### 수정 (`ir_transform.gl`)

추가 인자가 있으면 보간 경로 회피 (일반 printf 경로로):
```글
정수 첫인자 = 노드왼쪽풀[노드].
첫인자 >= 0이면 {
    정수 첫인자값노드 = 인자값풀[첫인자].
    정수 보간대상 = 0.
    정수 추가인자있음 = 0.
    인자다음풀[첫인자] >= 0이면 { 추가인자있음 = 1. }   (* ← 추가 *)
    (첫인자값노드 >= 0) * (노드종류풀[첫인자값노드] == 노드_문자열리터럴) * (추가인자있음 == 0) > 0이면 {
        ...
    }
}
```

### 정책 결정

- **순수 보간** (`"{이름}={값}"` 단독): 계속 작동
- **순수 형식** (`"값=%d", 값`): 계속 작동
- **혼용** (`"이름={이름} %d", 값`): 형식 지정자를 우선 처리하고 `{이름}`은 리터럴로 출력

사용자가 둘 중 하나를 선택하도록 강제 — 예측 가능성 > 편의성.

### 검증

```
이름={이름} 값=255 16진=ff   (* ← %d, %x 정상 *)
플레인: 철수=255              (* ← 순수 보간 그대로 작동 *)
```

### 공허 참조 함수 포인터

`공허 참조` 문법은 의도상 **void\* 일반 포인터**(예: `malloc` 결과)이므로 함수 포인터로 사용 불가가 맞음. 함수 포인터는 전용 문법 사용:

```글
[→공허] 콜백 = 함수이름.   (* void-반환 함수 포인터 *)
콜백().                     (* 정상 동작 *)
```

## 아키텍처 교훈

1. **의미 분리된 신호 필드 사용**: `임시변수원소크기풀`이 "배열이다"와 "포인터 원소크기" 두 신호를 겸용해서 포인터 스케일링 수정이 자체호스팅을 깨뜨림. 전용 필드 `임시변수포인터원소크기풀` 도입으로 해결.

2. **lvalue 경로의 독립성**: 주소 취득(`&`), 대입 좌변, 참조 역참조는 **값 계산(rvalue)과 다른 코드 경로**가 필요. 자식 노드를 `표현식변환`으로 일괄 처리하는 방식은 lvalue에 부적합.

3. **파서 우선순위 명시성**: 단항 연산자의 피연산자를 `기본표현식()`으로 받는 것은 C 스타일의 postfix > unary 우선순위와 충돌. 단항은 `접미표현식()` 결과에 적용해야 함.

4. **형태소 분리의 자기-참조 덫**: 의심 심볼을 등록하고 나서 "등록됐으니 유효"라고 판단하는 순환 논리가 크래시를 유발. 분리 허용 조건에 "어근이 접사 자체와 일치하지 않음" 같은 독립 검증 필요.
