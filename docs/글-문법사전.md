# 글 프로그래밍 언어 문법 사전

> 글 언어의 모든 문법 요소를 한 곳에서 관리하는 통합 사전.
> 각 요소의 정의, 용법, 상호작용, 한국어 자연스러움 평가를 포함한다.

---

## 1. 조사 (Particles) — 인자 역할 결정

한국어의 조사를 활용하여 함수 인자의 역할을 결정한다. 같은 글자도 문맥에 따라 다르게 해석될 수 있다.

### 1.1 조사 목록

| 조사 | 역할 | 설명 | 예시 | 비고 |
|------|------|------|------|------|
| 을 | 목적어 | 동작의 대상 (받침 있는 명사 뒤) | `책을 읽다` | |
| 를 | 목적어 | 동작의 대상 (받침 없는 명사 뒤) | `나를 보다` | `을`과 동일 역할 |
| 이 | 주어 | 동작의 주체 (받침 있는 명사 뒤) | `값이 크면` | 조건문에서 주로 사용 |
| 가 | 주어 | 동작의 주체 (받침 없는 명사 뒤) | `수가 0이면` | `이`와 동일 역할 |
| 은 | 주제 | 변수 선언/정의의 주제 (받침 있는 명사 뒤) | `결과는 42.` | 함수 정의에도 사용 |
| 는 | 주제 | 변수 선언/정의의 주제 (받침 없는 명사 뒤) | `이름은 "철수".` | `은`과 동일 역할 |
| 에 | 목적지 | 방향/위치 | `목록에 추가` | |
| 로 | 수단 | 방법/형변환 (받침 없는 명사 뒤) | `정수로 변환` | |
| 으로 | 수단 | 방법/형변환 (받침 있는 명사 뒤) | `문자열으로` | `로`와 동일 역할 |
| 에서 | 출처 | 출발점/원본 | `파일에서 읽기` | |
| 부터 | 시작 | 범위의 시작점 | `1부터` | 범위 반복에 사용 |
| 까지 | 종점 | 범위의 끝점 | `10까지` | 범위 반복에 사용 |
| 와 | 공동 | 함께/비교 대상 (받침 없는 명사 뒤) | `5와 같으면` | |
| 과 | 공동 | 함께/비교 대상 (받침 있는 명사 뒤) | `값과 같으면` | `와`와 동일 역할 |
| 의 | 소유 | 멤버 접근 | `학생의 이름` | C의 `.` 연산자 |
| 보다 | 비교 | 비교 기준 | `10보다 크면` | 비교 동사와 함께 |
| 에게 | 대상 | 간접 목적어 | `친구에게 보내다` | 현재 제한적 사용 |

### 1.2 조사의 다의성 (같은 글자, 다른 의미)

| 글자 | 문맥 1 | 문맥 2 | 구분 방법 |
|------|--------|--------|-----------|
| `이` | 주어 조사 (`값이 크면`) | 서술격 조사 (`42이다`) | 뒤에 오는 토큰으로 구분: `다`가 오면 서술격 |
| `의` | 멤버 접근 (`학생의 이름`) | 소유 표현 (`나의 값`) | 현재 동일 처리 — C `.` 연산자로 변환 |
| `가` | 주어 조사 (`값가 크면`) | 변수명 끝글자 | 심볼 테이블 우선: 등록된 이름이면 분리 안 함 |
| `자` | 어미 (`하자`) | 변수명 끝글자 (`대문자`) | 심볼 테이블 우선: 등록된 이름이면 분리 안 함 |

### 1.3 조사 분리 규칙

1. **심볼 테이블 우선**: 이미 선언된 변수/함수명이면 조사 분리하지 않음
2. **다중 바이트 우선**: `에서`(2글자)를 `에`+`서`(각 1글자)보다 먼저 검사
3. **분리 제외 사전**: `길이`, `나이`, `크기` 등 끝이 조사와 겹치는 단어는 보호
4. **최대 3단계 재귀**: `결과를` → `결과`+`를`, `크거나같으면` → `크거나같은`+`면`

## 2. 어미 (Endings) — 제어 흐름 결정

한국어의 어미를 활용하여 문장의 실행 방식을 결정한다.

### 2.1 어미 목록

| 어미 | 기능 | C 대응 | 사용 예시 | 구현 상태 |
|------|------|--------|-----------|-----------|
| -다 | 종결 | `;` | `쓰다.` | ✅ 완전 |
| -면 | 조건 | `if` | `크면 { }` | ✅ 완전 |
| -고 | 순차 연결 | `;` | `출력하고 증가하다.` | ✅ 완전 |
| -동안 | 반복 | `while` | `작은동안 { }` | ✅ 완전 |
| -거나 | 선택 | | `크거나같으면` | ⚠️ 비교동사 내에서만 |
| -서 | 인과 | | | ❌ 미구현 |
| -지만 | 양보 | | | ❌ 미구현 |
| -도록 | 목적 | | | ❌ 미구현 |
| -며/-면서 | 동시 | | | ❌ 미구현 |
| -라 | 명령 | | | ❌ 미구현 |
| -자 | 제안 | | | ❌ 미구현 |
| -까 | 의문 | | | ❌ 미구현 |

### 2.2 어미의 상호작용

```
동사어근 + 하 + 어미
         ↑       ↑
      투명 보조   제어 흐름
```

`하다`의 `하`는 투명 보조 동사로, 어미 앞에서 자동 제거된다:
- `계산하다` → `계산` + `하` + `다` → `계산` + `다` (하 제거)
- `계산하면` → `계산` + `하` + `면` → `계산` + `면`
- `계산하고` → `계산` + `하` + `고` → `계산` + `고`

## 3. 동사와 호출 (Verbs & Calls)

### 3.1 호출 방식

글 언어는 세 가지 호출 방식을 지원한다:

| 방식 | 문법 | 특징 | 예시 |
|------|------|------|------|
| SOV + 하다 | `인자를 함수하다.` | 가장 자연스러운 한국어 | `10을 피보나치하다.` |
| SOV + 다 | `인자를 함수다.` | 간결한 대안 | `10을 피보나치다.` |
| 직접 호출 | `함수(인자)` | C 스타일, 조사 무시 | `피보나치(10)` |

### 3.2 내장 동사 (Built-in Verbs)

컴파일러가 특별히 인식하는 동사들:

| 동사 | 기능 | C 대응 | 예시 |
|------|------|--------|------|
| 반환하다 | 값 반환 | `return 값;` | `결과를 반환하다.` |
| 증가하다 | 1 증가 | `값++;` | `번호를 증가하다.` |
| 감소하다 | 1 감소 | `값--;` | `번호를 감소하다.` |

### 3.3 현재 문법의 부자연스러운 패턴과 개선안

#### 문제 1: `-기하다` 패턴

**현재**: 함수명이 `-기` 형태(명사형)이면 `하다`를 붙일 때 어색해진다.

| 현재 (어색) | 자연스러운 한국어 | 함수명 | 문제 |
|-------------|------------------|--------|------|
| `쓰기하다` | `쓰다` | `쓰기` | `기하다`는 비문 |
| `읽기하다` | `읽다` | `읽기` | 동일 |
| `더하기하다` | `더하다` | `더하기` | 동일 |
| `빼기하다` | `빼다` | `빼기` | 동일 |

**해결 방향**: 동사 어근 ↔ 명사형(`-기`) 역변환 테이블

```
동사 어근    명사형(-기)    현재 호출        개선 호출
쓰          쓰기          쓰기하다         쓰다
읽          읽기          읽기하다         읽다
더하        더하기        더하기하다       더하다
빼          빼기          빼기하다         빼다
찾          찾기          찾기하다         찾다
열          열기          열기하다         열다
닫          닫기          닫기하다         닫다
만들        만들기        만들기하다       만들다
```

**구현 방법**: 렉서에서 `쓰다` → `쓰` + `다`로 분리한 후, `쓰`를 함수 검색할 때 `쓰기`도 후보로 포함

#### 문제 2: 함수명에 언더스코어가 포함된 복합 동사

| 현재 | 자연스러운 한국어 | 문제 |
|------|------------------|------|
| `파일_열기하다` | `파일을 열다` | 밑줄 함수는 `하다` 형태 불가 |
| `메모리_채우기하다` | `메모리를 채우다` | 동일 |
| `문자열_비교하다` | `문자열을 비교하다` | 이건 자연스러움 ✓ |

**참고**: 밑줄이 포함된 함수명은 직접호출(`함수(인자)`)만 가능하다는 기존 제약이 있다.

#### 문제 3: `번호++다` — 기호와 한국어 혼합

| 현재 | 대안 (이미 구현) | 비고 |
|------|------------------|------|
| `번호++다.` | `번호를 증가하다.` | 대안이 더 자연스러움 |
| `번호--다.` | `번호를 감소하다.` | 대안이 더 자연스러움 |
| `값 += 10다.` | (미구현) | `값에 10을 더하다` 형태 제안 |

## 4. 비교 동사 (Comparison Verbs)

한국어의 형용사/동사를 활용한 비교 표현. 관형사형과 연결형 활용을 지원한다.

### 4.1 비교 동사 활용표

| 기본형 | 연산자 | 관형사형 | 연결형(-면) | 사용 예시 |
|--------|--------|----------|-------------|-----------|
| 작다 | `<` | 작은 | 작으면 | `값이 10보다 작으면` |
| 크다 | `>` | 큰 | 크면 | `값이 10보다 크면` |
| 같다 | `==` | 같은 | 같으면 | `값이 0과 같으면` |
| 다르다 | `!=` | 다른 | 다르면 | `값이 0과 다르면` |
| 작거나같다 | `<=` | 작거나같은 | 작거나같으면 | `값이 10보다 작거나같으면` |
| 크거나같다 | `>=` | 크거나같은 | 크거나같으면 | `값이 10보다 크거나같으면` |

### 4.2 비교 구문 구조

```
[주어]이/가 [기준값]보다 [비교동사 관형사형]동안/면 { }
```

예시:
```글
번호가 100보다 작은동안 { }     → while (번호 < 100)
점수가 90보다 크거나같으면 { }  → if (점수 >= 90)
값이 0과 같으면 { }            → if (값 == 0)
```

## 7. 구문 구조 (Syntax Structures)

### 7.1 변수 선언

```글
타입 이름 = 초기값.            (* 타입 명시 *)
타입 이름.                     (* 초기값 없이 *)
이름은 초기값.                 (* 타입 추론 *)
타입 참조 이름.                (* 포인터 *)
타입[크기] 이름.               (* 배열 *)
상수 타입 이름 = 값.           (* 상수 *)
정적 타입 이름 = 값.           (* 정적 변수 *)
```

### 7.2 함수 정의

```글
(* 기본 형태 *)
[매개변수들 함수이름]은/는 → 반환타입 {
    본문
}

(* 매개변수 형식: 타입 이름조사 *)
[정수 값을 정수 기준에 비교하기]는 → 정수 { }
[정수 수를 피보나치]는 → 정수 { }
[시작하기]는 → 정수 { }
[시작하기]는 { }              (* 반환타입 생략 시 공허 *)
```

### 7.3 함수 호출

```글
(* SOV 호출 — 조사가 인자 역할 결정 *)
10을 피보나치하다.             (* 을→목적어, 하다→종결 *)
5를 3에 더하기하다.            (* 를→목적어, 에→목적지 *)
3에 5를 더하기하다.            (* 어순 자유 — 조사가 결정 *)

(* 직접 호출 — C 스타일 *)
피보나치(10).
더하기(5, 3).

(* 밑줄 함수는 반드시 직접 호출 *)
파일_열기("data.txt", "r").
```

### 7.4 제어 흐름

```글
(* 조건문 *)
조건이면 { }
조건이면 { } 아니면 { }
조건이면 { } 아니면 조건2이면 { } 아니면 { }

(* 비교 동사 조건 *)
값이 10보다 크면 { }
값이 0과 같으면 { }
값이 없으면 { }              (* NULL 검사 *)
값이 있으면 { }              (* 비-NULL 검사 *)

(* while 루프 *)
(조건)동안 { }
값이 100보다 작은동안 { }

(* C 스타일 for *)
반복 (정수 번호 = 0: 번호 < 10: 번호++) { }
(* 구분자는 : (쌍점) — C의 ; 대신 *)

(* do-while *)
반복하기 { } 조건 동안.

(* 범위 반복 *)
(1부터 10까지)동안 { }

(* 갈래 *)
갈래 (값) {
    경우 1: { }
    경우 2: { }
    기본: { }
}
```

### 7.5 구조체/나열/합침

```글
(* 구조체 — 쉼표+마침표 형식, 중괄호 아님 *)
묶음 학생은
    문자열 이름,
    정수 나이,
    정수 점수.

(* 열거형 *)
나열 색상은 {
    빨강 = 0,
    파랑,
    초록
}

(* 공용체 *)
합침 데이터는 정수 숫자, 실수 소수.
```

### 7.6 배열과 메모리

```글
(* 배열 선언 — 크기가 타입 뒤에 *)
정수[50] 점수목록.             (* C: int 점수목록[50]; *)
문자[256] 버퍼.

(* 동적 할당 *)
정수 참조 배열은 (100 * 크기(정수))을 할당하다.
배열[0] = 42.
배열을 해제하다.
```

## 9. 형태소 분석 규칙 (Morpheme Analysis)

### 9.1 접사 분리 순서

1. 다중 글자 조사/어미 먼저 (`에서`, `부터`, `까지`, `동안`, `면서` 등)
2. 단일 글자 조사/어미 (`을`, `를`, `이`, `가`, `은`, `는`, `에`, `로`, `다`, `면`, `고` 등)
3. 동사형 `하` 제거 (`계산하다` → `계산`+`다`)

### 9.2 분리 제외 사전

다음 단어들은 끝글자가 조사와 겹치지만 분리하지 않는다:

```
길이, 나이, 크기, 위치, 문자, 인자, 정자, 변자, 주소,
전체길이, 시작위치, 현재위치, 바이트수 ...
```

### 9.3 심볼 테이블 연동

- 선언된 변수/함수명은 조사 분리보다 우선
- `대문자`가 변수로 선언되면 `대문`+`자`로 분리하지 않음
- 미선언 식별자만 접사 분리 대상

## 10. 문법 개선 과제

### 10.1 구현 완료

| 개선 | 설명 | 예시 |
|------|------|------|
| SOV 반환 | 동사 위치에서 반환 | `결과를 반환하다.` |
| 동사형 증감 | 증가/감소 동사 인식 | `번호를 증가하다.` |
| `-고` 연결 | 문장 순차 연결 | `출력하고 증가하다.` |
| `하다` 동사형 | 동사어근 자동 분리 | `계산하다` → `계산`+`다` |

### 10.2 미구현 과제

| 과제 | 설명 | 난이도 | 영향도 |
|------|------|--------|--------|
| 동사 어근→명사형 역변환 | `쓰다`→`쓰기` 함수 검색 | 중 | 상 |
| `N번 반복하다` | 횟수 기반 반복 | 중 | 중 |
| 서술형 재대입 | `값은 100이다.` | 하 | 중 |
| 타입 추론 확장 | 함수 반환값 추론 | 상 | 상 |
| 동사형 복합 대입 | `값에 10을 더하다` → `값 += 10` | 중 | 중 |

### 10.3 자연스러움 등급

현재 문법의 한국어 자연스러움을 A(자연스러움)~D(부자연스러움)으로 평가:

| 구문 | 등급 | 현재 | 이상적 |
|------|------|------|--------|
| 비교 조건 | **A** | `값이 10보다 크면` | (이미 자연스러움) |
| 멤버 접근 | **A** | `학생의 이름` | (이미 자연스러움) |
| 변수 선언 | **A** | `정수 값은 42.` | (이미 자연스러움) |
| 반환문 | **A** | `결과를 반환하다.` | (이미 자연스러움) |
| 증감 | **B** | `번호를 증가하다.` | `번호를 늘리다` (더 구어적) |
| 반복 | **B** | `(조건)동안 { }` | `조건인동안 { }` |
| 함수 호출 | **C** | `10을 피보나치하다.` | `10의 피보나치를 구하다` |
| 출력 | **C** | `"안녕"을 쓰기하다.` | `"안녕"을 쓰다` |
| 복합 연산 대입 | **D** | `값 += 10다.` | `값에 10을 더하다` |
| 기호 증감 | **D** | `번호++다.` | `번호를 증가하다` |

## 12. 다의성 해결 규칙 (Disambiguation)

같은 글자가 문맥에 따라 다른 의미를 갖는 경우의 해결 규칙.
이 섹션은 컴파일러의 형태소분석기가 올바른 판단을 하기 위한 핵심 참조이다.

### 12.1 조사 vs 단어 일부

| 글자 | 조사로 해석 | 단어 일부로 해석 | 해결 규칙 |
|------|-------------|------------------|-----------|
| 이 | 주어 조사 (`값이 크면`) | 단어 끝글자 (`나이`, `길이`, `높이`) | 제외사전 > 심볼테이블 > 접사분리 |
| 가 | 주어 조사 (`수가 0이면`) | 단어 끝글자 (`인가`, `평가`) | 제외사전 > 심볼테이블 > 접사분리 |
| 자 | 어미 (`하자`) | 단어 끝글자 (`문자`, `인자`, `글자`, `숫자`) | 제외사전 > 심볼테이블 > 접사분리 |
| 과 | 공동 조사 (`값과 비교`) | 단어 끝글자 (`결과`, `효과`, `통과`) | 제외사전 > 심볼테이블 > 접사분리 |
| 로 | 수단 조사 (`정수로`) | 단어 끝글자 (`가로`, `세로`, `경로`, `매크로`) | 제외사전 > 심볼테이블 > 접사분리 |
| 서 | 어미 (`해서`) | 단어 끝글자 (`문서`, `커서`, `순서`) | 제외사전 > 심볼테이블 > 접사분리 |
| 면 | 조건 어미 (`크면`) | 단어 끝글자 (`화면`, `장면`, `표면`) | 제외사전 > 심볼테이블 > 접사분리 |
| 고 | 순차 어미 (`하고`) | 단어 끝글자 (`보고`) | 제외사전 > 심볼테이블 > 접사분리 |
| 의 | 소유 조사 (`학생의`) | 단어 끝글자 (`거의`) | 제외사전 > 심볼테이블 > 접사분리 |

**해결 우선순위** (높은 것부터):
1. 제외사전에 등록된 단어이면 → 분리하지 않음
2. 심볼테이블에 등록된 변수/함수명이면 → 분리하지 않음
3. 어근이 키워드이면 → 분리
4. 어근이 알려진 단어이면 → 분리
5. 그 외 → 통째로 식별자

### 12.2 `이` 의 세 가지 역할

| 문맥 | 역할 | 예시 | 처리 |
|------|------|------|------|
| 명사 + 이 + 동사/형용사 | 주어 조사 | `값이 크면` | 접사분리: `값`+`이` |
| 숫자/명사 + 이 + 다 | 서술격 조사 | `42이다`, `학생이다` | `이`+`다` 분리 후 서술격 처리 |
| 단어 내부 | 단어 일부 | `길이`, `나이` | 제외사전으로 보호 |

### 12.3 `다` 의 두 가지 역할

| 문맥 | 역할 | 예시 | 처리 |
|------|------|------|------|
| 동사/함수 + 다 | 종결 어미 | `쓰기다.`, `반환하다.` | 문장 종결 |
| 비교동사 기본형 | 동사 자체 | `크다`, `작다`, `같다` | 키워드로 인식 (비교 연산자) |

### 12.4 `하` 의 처리

| 문맥 | 처리 | 예시 | 결과 |
|------|------|------|------|
| 함수명 + 하 + 어미 | `하` 제거, 어미만 분리 | `계산하다` → `계산`+`다` | 함수 호출 + 종결 |
| 함수명 + 하 + 고 | `하` 제거, 연결 | `출력하고` → `출력`+`고` | 함수 호출 + 순차 |
| 함수명 + 하 + 면 | `하` 제거, 조건 | `성공하면` → `성공`+`면` | 함수 호출 + 조건 |

### 12.5 `&` 의 두 가지 의미

| 문맥 | 의미 | 예시 |
|------|------|------|
| 단항 접두사 | 주소 연산자 | `&변수` |
| 이항 연산자 | 비트 AND | `값 & 마스크` |

파서가 문맥(단항/이항 위치)으로 구분.

## 13. 동사 활용 체계 (Verb Conjugation)

글 언어에서 함수 호출은 한국어 동사 활용 체계를 따른다.

### 13.1 현재 동사 활용 패턴

```
함수명 + 하 + 어미    (하다 동사형)
함수명 + 어미          (직접 어미 접속)
```

| 함수명 | +다 (종결) | +면 (조건) | +고 (순차) | +동안 (반복) |
|--------|-----------|-----------|-----------|------------|
| 쓰기 | 쓰기다 / 쓰기하다 | 쓰기하면 | 쓰기하고 | — |
| 피보나치 | 피보나치다 / 피보나치하다 | 피보나치하면 | 피보나치하고 | — |
| 증가 | 증가하다 | 증가하면 | 증가하고 | — |
| 반환 | 반환하다 | — | — | — |

### 13.2 문제: `-기하다` 비문

한국어에서 `-기` 명사형에 `하다`를 붙이면 비문이다:

| 비문 (현재) | 자연스러운 한국어 | 함수 이름 |
|-------------|------------------|-----------|
| 쓰기하다 | **쓰다** | 쓰기 |
| 읽기하다 | **읽다** | 읽기 |
| 더하기하다 | **더하다** | 더하기 |
| 빼기하다 | **빼다** | 빼기 |
| 찾기하다 | **찾다** | 찾기 |
| 열기하다 | **열다** | 열기 |
| 닫기하다 | **닫다** | 닫기 |
| 만들기하다 | **만들다** | 만들기 |

### 13.3 해결안: 동사어근 ↔ 명사형 역변환

`쓰다`로 호출하면 `쓰기` 함수를 찾는 역변환 테이블:

```
동사어근 → 명사형(-기) 변환 규칙:
  어근 + "기" = 명사형

역변환 (호출 시):
  "쓰" + "다" → 함수 "쓰" 검색 실패 → "쓰기" 검색 → 성공
```

**필요한 컴파일러 변경**:
- 형태소분석기: `쓰다` → `쓰`+`다` 분리 (현재도 작동)
- 구문분석기: SOV 동사 위치에서 함수명 검색 시, `어근`으로 못 찾으면 `어근+기`로 재검색

### 13.4 표준 라이브러리 동사 활용 대응표

| 현재 호출 (어색) | 개선 호출 (자연스러움) | 함수명 | 변환 규칙 |
|------------------|----------------------|--------|-----------|
| `"안녕"을 쓰기하다.` | `"안녕"을 쓰다.` | 쓰기 | 쓰→쓰기 |
| `파일을 열기하다.` | `파일을 열다.` | 열기 (직접호출만) | 열→열기 |
| `5를 3에 더하기하다.` | `5를 3에 더하다.` | 더하기 | 더하→더하기 |
| `배열을 해제하다.` | `배열을 해제하다.` | 해제 | (이미 자연스러움) |
| `100을 할당하다.` | `100을 할당하다.` | 할당 | (이미 자연스러움) |

**참고**: `해제`, `할당`, `반환`, `증가`, `감소` 등 `-기`로 끝나지 않는 함수는 현재도 자연스럽다.
문제는 `-기`로 끝나는 함수명에만 해당된다.

## 14. 분리 제외 사전 (Protected Words)

끝글자가 조사/어미와 겹치지만 분리하면 안 되는 단어 목록.

### 14.1 현재 등록된 제외 단어

| 끝글자 | 제외 단어들 |
|--------|-------------|
| 이 | 나이, 사이, 넓이, 높이, 길이, 깊이, 무늬 |
| 자 | 인자, 글자, 숫자, 문자, 전자, 원자, 양자 |
| 로 | 가로, 세로, 경로, 매크로, 제로, 주로 |
| 서 | 문서, 커서, 선서, 순서 |
| 과 | 결과, 효과, 통과 |
| 가 | 인가, 평가 |
| 면 | 화면, 장면, 표면 |
| 의 | 거의 |
| 고 | 보고 |
| 기타 | 합침, 인쇄 |

### 14.2 sub-word 보호 규칙

복합어의 끝부분이 제외사전 단어를 형성하면 분리하지 않는다:
- `텍스트길이` → `텍스트길`+`이`(✗) → `길이`가 제외단어 → 분리 안 함(✓)
- `전체길이` → `전체길`+`이`(✗) → `길이`가 제외단어 → 분리 안 함(✓)
- `시작위치` → `시작위`+`치`(✗) → `치`는 접사 아님 → 해당 없음

### 14.3 밑줄 보호 규칙

밑줄(`_`)이 포함된 식별자는 접사 분리하지 않는다:
- `파일_열기` → 분리 안 함 (밑줄 보호)
- `정수목록_추가` → 분리 안 함
