글 언어로 글 컴파일러를 작성하여, C 의존성 없이 독립적으로 동작하는 프로그래밍 언어 생태계를 구축합니다.

자체호스팅이 성공적으로 완료되었습니다.

- 컴파일러 소스 7개 파일, ~4100줄 — 100% 글 언어

- SHA256: `0C122A963AD3E951E919D9B1D0687430E1B93511506C25B23874BA0FC96385A9`

## Phase 3: 자체 호스팅 — ✓ 완료

### 3.1 — 핵심 자료구조 (글로 작성) ✓

SoA(Struct-of-Arrays) 패턴으로 구현:
```
정수[8192] 노드종류풀.
문자열[8192] 노드이름풀.
정수[8192] 노드왼쪽풀.
정수[8192] 노드오른쪽풀.
```

### 3.2 — 형태소 분석기 (글로 작성) ✓

- UTF-8 디코더 구현
- 한글 코드포인트 판별
- 키워드 테이블
- 3단계 재귀 접사 분리 + 2.5단계 서술격 조사 "이" 패턴

### 3.3 — 구문 분석기 (글로 작성) ✓

- 재귀 하향 파서 포팅
- SOV 호출 파싱 + Pratt 파서
- SoA 기반 AST 생성

### 3.4 — 코드 생성기 (글로 작성) ✓

C11 코드 생성 유지. 모든 표현식/문장/선언 지원.

### 3.5 — 부트스트래핑 완료 ✓

1. C 부트스트랩 → 글 컴파일러 소스 컴파일 → Stage 0 (257024 bytes)
2. Stage 0으로 글 컴파일러 소스 다시 컴파일 → Stage 1 (257024 bytes)
3. Stage 1으로 글 컴파일러 소스 다시 컴파일 → Stage 2 (257024 bytes)
4. **Stage 1 == Stage 2 (SHA256 일치)** → 자체호스팅 완료

### 해결한 주요 이슈

| 이슈 | 원인 | 해결 |
|------|------|------|
| 포함 전처리 헥스이스케이프 | 부트스트랩 코드생성기 이중 이스케이프 | 실제 한글 "포함"으로 교체 |
| 포함 키워드 오탐 | 소스 중간에서 "포함" 매칭 | 줄 시작에서만 매칭 |
| argv ANSI/UTF-8 | Windows CP949 인코딩 | `_ansi_to_utf8` 런타임 헬퍼 |
| main(void) argv | 매개변수 없는 main에서 argv 참조 | 조건부 출력 |
| 2.5단계 누락 | "이면"/"이다"에서 "이" 미분리 | 서술격 조사 분리 로직 추가 |
| 나머지 연산자 | `%%` 이중 출력 | `%`로 수정 |
