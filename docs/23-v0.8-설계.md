## 후보 4 — lexer/parser interface 재설계

### 핵심 아이디어

parser가 raw 합성어 token을 받고 *자체적으로* 분리/등록. lexer는 *stateful flag 없이* 단순 토큰 emit. 즉:
- **lexer**: 단어 경계만 식별. 모든 한글 단어를 *그대로* `종류_단어` 토큰으로 emit (조사 분리 시도 X)
- **parser**: context 따라 단어를 분리 또는 통째 사용. 예:
  - 변수 선언 context: 단어 통째로 변수 이름
  - 함수 호출 context: 끝 조사를 분리해 인자 역할로 인식
  - 매개변수 선언 context: 끝 조사를 분리해 역할 추출

### 외부 영향

- **사용자 syntax 변경 0**: 기존 글 코드 그대로 컴파일
- **글 본가 src 변경 0**: 자체호스팅 src도 그대로
- **lexer/parser 코드 대공사**: lexer ~1200 LOC + parser 일부 + IR transform 일부

## v0.7.4 현재 parser context 진입점 매핑

`src/parser.gl`에서 `선언이름모드 = 1` 설정 위치 6곳 (v0.7.4 시점):

| 위치 | 라인 | Context | lexer가 받는 단어 의도 |
|---|---|---|---|
| 변수 선언 | 980 | `정수 X = 값.` | X = 변수 이름 (통째) |
| for 루프 변수 | 1222 | `반복 (정수 i = 0 ...)` | i = 변수 이름 |
| struct 필드 | 1582 | `묶음 P { 정수 x. }` | x = 필드 이름 |
| **매개변수** | 1634 | `[정수 X로 함수]` | X = 매개 이름 + `로` = 역할 (분리 필수) |
| 외부 변수 | 1707 | `외부 정수 X.` | X = 변수 이름 |
| union 필드 | 1784 | `합침 U { 정수 x. }` | x = 필드 이름 |

**관찰**: 매개변수 한 곳만 *조사 분리 필요*. 나머지 5곳은 *통째 식별자* 의도. lexer는 모든 6곳을 `선언이름모드=1`로만 신호 받음 → 구분 불가 → stage 1.7의 복잡 분기.

함수 호출 시 인자에 조사가 붙음 (`5를 10에 더하기다.`): lexer가 `5를`/`10에` 단어를 자동 분리하여 `5`+`를`/`10`+`에` 토큰 emit. parser는 분리된 토큰 받음.

## 새 lexer/parser interface 설계

### 원칙

1. **lexer는 상태 없음** — `선언이름모드` 등 flag 제거
2. **lexer는 단어 경계 + 키워드 매칭만** — 조사 분리 시도 X
3. **parser가 raw 단어에서 분리 결정** — context 따라

### lexer 새 출력

단어 처리 흐름이 다음 3개 분기로 단순화:

```
한글 단어 읽기
  ├── 전체가 키워드 (`정수`, `이면`, `반환` 등 41개) → 키워드 토큰
  ├── 전체가 접사 (`을`, `를`, `에서` 등 32개) → 접사 토큰
  └── 그 외 → 종류_한글단어 토큰 (raw)
```

stage 1.5/1.6/1.7/2/2.3/2.5/3 모두 **제거**. lexer ~1200 LOC → 추정 ~600 LOC.

### parser 책임 추가

`종류_한글단어` 토큰을 받으면 *context*에 따라:

| Context | 처리 |
|---|---|
| 변수/필드/외부 선언 후 | 단어 통째로 식별자 이름 등록 |
| 매개변수 선언 후 | 단어 끝에서 접사 분리 시도. 끝이 접사 글자면 어근(이름) + 접사(역할) |
| 함수 호출 인자 위치 | 단어 끝에서 접사 분리. 어근이 인자, 접사가 역할 |
| 기타 (산술/대입 등) | 일반 식별자로 등록 또는 lookup |

### 신규 헬퍼 함수 (lexer.gl, parser에서 호출 가능)

```글
(* 단어 끝에서 접사 분리 시도. 분리 시 토큰 버퍼에 push, 통째 시 0 반환. *)
[문자열 단어를 정수 길이를 단어분리시도]는 → 정수 {
    (* 분리 결과를 lexer 토큰 버퍼(버퍼에넣기)에 직접 push *)
    (* 반환: 0 = 분리 안 됨, push 토큰 수 = 분리됨 *)
}
```

### 분리 결과 자료구조 결정 (2026-05-16)

**채택: 토큰 버퍼 직접 push** (lexer 기존 stage들과 동일 패턴)

- 헬퍼가 `버퍼에넣기(...)` 호출로 분리된 토큰들을 버퍼에 push
- parser는 다음 토큰 read 시 push된 토큰들 순차로 받음
- 반환값: 0 (분리 안 됨, parser가 단어 통째 사용) / 1+ (push된 토큰 수)

대안 검토:
- 전역 변수에 결과 저장 — parser가 token vs 분리결과 두 path 처리 필요, 흐름 분리됨 → 거름
- out 포인터 — 글 syntax에서 어려움 → 거름

이 헬퍼는 lexer stage 2와 동일한 부작용(버퍼 push)을 갖되, **호출 시점이 parser context aware**. v0.8 작업의 핵심 추상화.

### 점진적 이식 계획

1. ✓ 스텁 정의 (현재)
2. → stage 2 기본 동작 이식 (단어 끝 접사 매칭, 어근=키워드 시 분리)
3. → stage 2.3 (동사형) 이식
4. → stage 2.5 (서술격 이) 이식
5. → stage 3 (어근 내부 재귀) 이식
6. → stage 1.5/1.6/1.7 점진 단순화 (parser가 호출하는 케이스로 흡수)
7. → 옛 lexer 흐름의 stage 2~3 제거 (헬퍼가 대체)
8. → parser에 매개변수/호출 context별 헬퍼 호출 시점 결정
9. → lexer 단순화 완료, R-02 해결

### 토큰 종류 추가

- `종류_한글단어` 신규: raw 한글 식별자 단어

기존 `종류_식별자`는 유지 (parser가 분리 후 emit 또는 직접 사용).

## 1. 배경

v0.7.4까지 글 lexer는 식별자에 1바이트 조사(이/가/와/과/로/의/에) 글자가 끝에 오면 자동 분리 시도. 다음 두 정상 syntax가 동시 지원:

```글
(* 분리되지 않아야 정상 *)
정수 블록와이 = 42.        (* 변수 이름이 와+이 패턴 *)

(* 분리되어야 정상 *)
정수 읽은길이는 strlen(x).  (* X + 는 = X = strlen(x) 등가 *)
```

lexer는 두 케이스를 구분할 정보가 없음 — 둘 다 어근(`블록와`/`읽은길이`) + 1바이트 접사(`이`/`는`) 패턴. v0.7.4 spike 3회 모두 자체호스팅과 충돌. 글 syntax 자체의 모호성이 근본 원인.

## 2. 채택 결정 (2026-05-16)

**후보 2 + 노선 A**: 변수 초기화에서 공백 분리 강제. 매개변수 syntax 그대로.

### 새 정상 syntax

```글
(* 변수 선언 — 다음 셋만 정상 *)
정수 X.                    (* 선언만 *)
정수 X = 값.               (* 초기화 *)
정수 X 는 값.              (* 한국어 초기화, 공백 필수 *)

(* 매개변수 — 그대로 (조사 자동 분리 유지) *)
[정수 X를 정수 Y로 함수]은 → 정수 { ... }
```

### Deprecated syntax

```글
(* v0.8부터 오류 *)
정수 X는 값.              (* 공백 없는 조사 — deprecated *)
정수 블록와이는 42.       (* 합성어 + 조사 — deprecated *)
```

| 7곳 외에 숨은 의존 식별자 발견 | 중 | 빌드 실패 시 trace 재추가하여 식별자 카탈로그 갱신, 추가 마이그레이션 |
| lexer 변경이 매개변수 외 영향 | 중 | 매개변수선언모드 flag로 좁게 적용. 자체호스팅 고정점이 즉시 검증 |
| 외부 사용자 코드(globally `정수 X는` 사용) 영향 | 낮 | v0.8 changelog 명시. 마이그레이션 가이드 |
| 자체호스팅 깨짐 (v0.7.4 시도 3 재현) | 중 | 7곳 마이그레이션 *먼저* 한 뒤 lexer 변경 — v0.7.4 시도와 반대 순서 |

## 7. 핵심 차이점 (v0.7.4 시도 3과의 비교)

v0.7.4의 시도 3에서 `매개변수선언모드` flag 도입했으나 **글 본가 코드는 그대로** 두고 lexer만 변경 → 자체호스팅 깨짐.

v0.8 = **글 본가 7곳을 먼저 마이그레이션 + 그 다음 lexer 변경**. 자체호스팅 자기 코드가 새 syntax로 변경됐으니 lexer가 분리 안 해도 정상.
