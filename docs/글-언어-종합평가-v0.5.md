# 글 프로그래밍 언어 — 종합 평가 및 소개서 (v0.5)

---

## 한 줄 소개

> **글(Geul)**은 한국어의 교착어 문법을 프로그래밍 문법으로 채용한 세계 유일의 **자체호스팅·네이티브 코드 생성 한글 프로그래밍 언어**입니다.

---

## 1. 글 언어란?

글은 프로그래밍의 모든 요소를 한국어로 표현하는 독자적 프로그래밍 언어입니다.

```글
포함 "std.gl"

[정수 수를 피보나치]는 → 정수 {
    수 <= 1이면 { 반환 수. }
    정수 앞은 (수 - 1)을 피보나치다.
    정수 뒤는 (수 - 2)를 피보나치다.
    반환 앞 + 뒤.
}

[시작하기]는 {
    "피보나치(10) = %d\n"을 10을 피보나치다를 쓰다.
}
```

영어 키워드가 단 하나도 없습니다. `if`는 `이면`, `while`은 `동안`, `return`은 `반환`, `struct`는 `묶음`입니다. 한국어를 아는 사람이라면 코드를 **읽는 것만으로** 프로그램의 동작을 이해할 수 있습니다.

## 3. 핵심 차별점

### 3.1 한국어 교착어 문법 = 프로그래밍 문법

글의 가장 독특한 특징은 한국어의 **교착어 문법**을 그대로 프로그래밍 문법으로 사용한다는 점입니다.

- **조사가 인자의 역할을 결정합니다**
  ```글
  5를 3에 더하다.        (* 5 + 3 — '를'은 목적어, '에'는 목표 *)
  "이름: %s\n"을 이름을 쓰다.  (* printf — '을'로 인자 전달 *)
  ```

- **어미가 제어 흐름을 결정합니다**
  ```글
  값 > 0이면 { }       (* if — '이면' 어미 *)
  (조건)동안 { }        (* while — '동안' 어미 *)
  결과를 반환하다.       (* return — '하다' 어미로 동사 실행 *)
  ```

- **SOV(주어-목적어-동사) 어순**을 따릅니다
  ```글
  10을 피보나치하다.     (* 한국어 어순: 대상을 동사하다 *)
  ```

다른 비영어 언어들은 키워드만 번역한 수준이지만, 글은 **문법 구조 자체**가 한국어입니다.

### 3.2 MSVC/GCC/LLVM 없이 독립 실행

글 컴파일러는 `.글` 소스 파일을 읽어 **x86-64 기계어를 직접 생성**하고 **PE(Windows 실행 파일) 형식으로 직접 기록**합니다. 중간에 C 컴파일러, 어셈블러, 링커가 개입하지 않습니다.

```
소스.글 → [글 컴파일러] → 소스.exe (바로 실행 가능)
```

이것은 Go와 Zig 수준의 독립성입니다. 사용자는 컴파일러 파일 하나와 표준 라이브러리 파일 하나만 있으면 프로그램을 작성하고 실행할 수 있습니다.

### 3.3 자체호스팅 — 자기 자신을 컴파일하는 컴파일러

글 컴파일러(14,265줄)는 글 언어로 작성되어 있으며, **자기 자신을 컴파일**할 수 있습니다.

```
컴파일러 A → [A로 소스 컴파일] → 컴파일러 B
컴파일러 B → [B로 소스 컴파일] → 컴파일러 C
SHA256(B) == SHA256(C)  ← 고정점 달성!
```

이것은 컴파일러의 **정확성에 대한 가장 강력한 증명**입니다. 컴파일러가 자신의 복잡한 코드를 올바르게 처리할 수 있다는 것은, 다른 프로그램도 올바르게 컴파일할 수 있다는 신뢰를 줍니다.

### 3.4 문자열 보간

별도의 함수 호출 없이 변수를 문자열에 직접 삽입합니다.

```글
문자열 이름 = "철수".
정수 나이 = 25.
"{이름}은 {나이}세입니다\n"을 쓰다.
```

출력: `철수은 25세입니다`

## 5. 강점

### ✅ 언어 설계
- **한국어 원어민에게 직관적** — 코드가 한국어 문장처럼 읽힘
- **교착어 문법의 프로그래밍 활용** — 세계 최초 수준의 시도
- **문자열 보간** — `{변수}`로 간편한 출력
- **동사형 호출** — `값을 함수하다` 한국어 어순

### ✅ 기술
- **완전한 자체호스팅** — 고정점 SHA256 증명
- **네이티브 코드 직접 생성** — LLVM/GCC 의존 없음
- **단일 파일 배포** — 컴파일러 1개 + 표준라이브러리 1개
- **빠른 컴파일** — 11,000줄을 11ms에 컴파일

### ✅ 실용성
- **VS Code 통합** — 문법 강조, F5 실행, 스니펫
- **한 줄 설치** — PowerShell 명령으로 즉시 설치
- **C 상호운용** — printf, malloc 등 C 표준 함수 직접 호출 가능

## 6. 약점 및 현재 한계

### ⚠️ 플랫폼
- **Windows 전용** — Linux/macOS 미지원 (ELF 생성기 골격만 존재)
- **x86-64 전용** — ARM 등 다른 아키텍처 미지원

### ⚠️ 최적화
- **기본적 최적화만** — 인라이닝, 루프 최적화 등이 구현되어 있으나 안정성을 위해 비활성
- **레지스터 할당 없음** — 모든 변수가 스택 메모리 사용
- **C /Od 대비 ~2x 느림** — 최적화 없는 C 수준

### ⚠️ 생태계
- **패키지 관리자 없음** — 외부 라이브러리 관리 체계 미비
- **디버그 정보 없음** — PDB 미생성, WinDbg 심볼 없음
- **에러 처리 부재** — try/catch 같은 구조 없음
- **다중 파일 컴파일 제한** — 헤더 생성 미지원
- **커뮤니티 초기 단계** — 사용자 기반 미형성

### ⚠️ 언어
- **변수명 제약** — `자`로 끝나는 이름은 어미 분리 충돌 가능
- **C 런타임 의존** — msvcrt.dll 함수를 직접 호출

## 8. 소개 가이드 — 글 언어를 이렇게 설명하세요

### 한 줄 버전
> "한국어로 코드를 작성하면 바로 실행 파일이 만들어지는 프로그래밍 언어입니다."

### 30초 버전
> "글은 한국어의 조사와 어미를 프로그래밍 문법으로 사용하는 언어입니다. `값을 함수하다`처럼 한국어 문장 그대로 코드를 작성합니다. 컴파일러 자체도 글 언어로 작성되어 있고, 자기 자신을 컴파일할 수 있습니다. Visual Studio나 GCC 같은 외부 도구 없이 `.글` 파일을 바로 `.exe`로 만들어줍니다."

### 2분 버전
> "프로그래밍 언어의 키워드는 대부분 영어입니다. `if`, `for`, `return` — 비영어권 프로그래머에게는 이것이 진입 장벽입니다. 중국어(文言), 일본어(なでしこ), 아랍어(قلب) 같은 비영어 프로그래밍 언어가 있지만, 모두 인터프리터나 트랜스파일러 수준입니다.
>
> 글은 다릅니다. 한국어의 교착어 특성 — 조사(`을/를`, `에`, `로`)가 인자 역할을, 어미(`이면`, `동안`, `하다`)가 제어 흐름을 결정합니다. 이것은 단순한 키워드 번역이 아니라 **문법 구조 자체가 한국어**인 것입니다.
>
> 기술적으로도 특별합니다. 글 컴파일러는 글 언어로 작성된 14,000줄의 프로그램이며, 자기 자신을 컴파일하여 바이트 단위로 동일한 바이너리를 생성합니다 — 이것을 '고정점'이라고 합니다. 비영어 프로그래밍 언어 중 이 수준에 도달한 것은 글이 유일합니다.
>
> 실용성도 갖추었습니다. VS Code에서 `.글` 파일을 열면 문법 강조가 적용되고, F5를 누르면 컴파일·실행됩니다. 설치는 PowerShell 한 줄이면 됩니다."
