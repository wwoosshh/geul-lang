# 글 언어 - 컴파일 상세 흐름

## 완전한 컴파일 예제: 피보나치 수열

### 원본 소스 (피보나치.글)

```글
포함 "표준/입출력"

[정수 번째의 피보나치를 구하기]는 → 정수
    번째가 1보다 작거나같으면
        번째를 반환하다.
    번째에서 1을 빼서 피보나치를 구하고
    번째에서 2를 빼서 피보나치를 구하여
    합하여 반환하다.

[시작하기]는
    정수 번호가 0부터 9까지동안
        번호의 피보나치를 구하여 출력하다.
```

---

### 단계 1: 전처리

```
입력: 소스 텍스트

처리:
1. UTF-8 BOM 제거 (있으면)
2. NFC 정규화 (모든 한글을 조합형으로 통일)
3. "포함 "표준/입출력"" → 표준/입출력.글 파일 내용 삽입
4. 소스 위치 테이블 생성: { 파일: "피보나치.글", 줄: N, 열: M }

출력: 전처리된 소스 텍스트 + 소스위치맵
```

---

### 단계 2: 형태소 분석 (어휘 분석)

```
입력: 전처리된 소스 텍스트

처리 과정 (줄별):

줄 3: "[정수 번째의 피보나치를 구하기]는 → 정수"
  [     → 토큰(여는대괄호)
  정수   → 토큰(타입_정수)        [키워드 테이블에서 매칭]
  번째   → 토큰(식별자, "번째")
  의     → 토큰(조사_의)          [1음절 조사 매칭]
  피보나치 → 토큰(식별자, "피보나치")
  를     → 토큰(조사_를)
  구하기  → 토큰(식별자, "구하기")  [동사원형]
  ]     → 토큰(닫는대괄호)
  는     → 토큰(조사_는)          [주제 설정]
  →     → 토큰(화살표)
  정수   → 토큰(타입_정수)

줄 4: "번째가 1보다 작거나같으면"
  번째   → 토큰(식별자, "번째")
  가     → 토큰(조사_가)          [주어 조사]
  1     → 토큰(정수_리터럴, 1)
  보다   → 토큰(조사_보다)        [비교 조사]
  작거나같으 → 토큰(비교_작거나같다) [비교 키워드]
  면     → 토큰(어미_면)          [조건 어미]

줄 5: "번째를 반환하다."
  번째   → 토큰(식별자, "번째")
  를     → 토큰(조사_를)
  반환하  → 토큰(키워드_반환) + 동사화("하")
  다     → 토큰(어미_다)          [종결 어미]
  .     → 토큰(마침표)

줄 6: "번째에서 1을 빼서 피보나치를 구하고"
  번째   → 토큰(식별자, "번째")
  에서   → 토큰(조사_에서)        [출처 조사, 2음절 먼저 매칭]
  1     → 토큰(정수_리터럴, 1)
  을     → 토큰(조사_을)
  빼     → 토큰(식별자, "빼")     [동사어근]
  서     → 토큰(어미_서)          [파이프라인 어미]
  피보나치 → 토큰(식별자, "피보나치")
  를     → 토큰(조사_를)
  구하   → 토큰(식별자, "구하")    [동사어근]
  고     → 토큰(어미_고)          [순차 접속 어미]

출력: 토큰 스트림 (각 토큰에 소스 위치 첨부)
```

**형태소 분리 알고리즘 상세**:

```
함수 한글토큰분리(텍스트):
    결과 = []
    위치 = 0

    반복 (위치 < 텍스트.길이):
        // 공백 건너뛰기
        만약 공백이면: 위치++; 계속

        // 숫자 리터럴
        만약 숫자이면: 숫자토큰읽기(); 계속

        // 문자열 리터럴
        만약 '"'이면: 문자열토큰읽기(); 계속

        // 기호/연산자
        만약 기호이면: 기호토큰읽기(); 계속

        // 한글 토큰 (핵심!)
        만약 한글이면:
            단어 = 연속한글읽기()     // 공백이나 비한글까지 읽기

            // 1. 키워드 완전 매칭 시도
            만약 키워드테이블[단어]:
                결과.추가(키워드토큰(단어))
                계속

            // 2. 뒤에서부터 조사/어미 분리 시도
            (어근, 접사) = 접사분리(단어)

            만약 접사 != 없음:
                // 어근이 키워드이거나 심볼테이블에 있으면 분리 확정
                만약 키워드테이블[어근] 또는 심볼테이블[어근]:
                    결과.추가(토큰(어근))
                    결과.추가(접사토큰(접사))
                    계속

                // 동사형 확인: "~하" + 어미
                만약 어근이 "하"로 끝남:
                    동사어근 = 어근에서 "하" 제거
                    결과.추가(동사토큰(동사어근))
                    결과.추가(접사토큰(접사))
                    계속

            // 3. 분리 실패: 전체를 식별자로 처리
            결과.추가(식별자토큰(단어))

    반환 결과

함수 접사분리(단어):
    // 긴 접사부터 매칭 (최장 매칭)
    접사목록 = [
        // 3음절
        "거나같으",
        // 2음절
        "에서", "부터", "까지", "보다", "으로", "에게",
        "거나", "지만", "도록", "동안", "면서",
        // 1음절
        "을", "를", "이", "가", "은", "는",
        "에", "로", "와", "과", "의",
        "다", "면", "고", "서", "라", "까", "며",
    ]

    각 접사에 대해:
        만약 단어가 접사로 끝나면:
            어근 = 단어[0 : 길이-접사길이]
            만약 어근길이 > 0:
                반환 (어근, 접사)

    반환 (단어, 없음)
```

### 단계 3: 구문 분석 (파서)

```
입력: 토큰 스트림

파서 타입: 재귀하강 파서 + 프랫 파싱 (표현식)

출력 AST:

프로그램노드
├── 함수정의노드 "구하기"
│   ├── 매개변수: [(정수, "번째", 의), (없음, "피보나치", 를)]
│   ├── 반환타입: 정수
│   └── 본문:
│       ├── 조건노드
│       │   ├── 조건: 비교노드
│       │   │   ├── 주어: 식별자("번째"), 조사: 가
│       │   │   ├── 대상: 리터럴(1), 조사: 보다
│       │   │   └── 비교: 작거나같다
│       │   ├── 참: 블록
│       │   │   └── 반환노드
│       │   │       └── 식별자("번째")
│       │   └── 거짓: 없음
│       └── 파이프라인노드
│           ├── 단계1: 호출("빼", [("번째",에서), (1,을)]) -서→
│           ├── 단계2: 호출("구하", [(_,없음), ("피보나치",를)]) -고→
│           ├── 단계3: 호출("빼", [("번째",에서), (2,을)]) -서→
│           ├── 단계4: 호출("구하", [(_,없음), ("피보나치",를)]) -여→
│           ├── 단계5: 호출("합하", []) -여→
│           └── 반환
└── 함수정의노드 "시작하기"
    ├── 매개변수: []
    ├── 반환타입: 공허
    └── 본문:
        └── 범위반복노드
            ├── 변수: "번호"
            ├── 시작: 0 (부터)
            ├── 끝: 9 (까지)
            └── 본문: 파이프라인
                ├── 호출("구하", [("번호",없음), ("피보나치",를)]) -여→
                └── 호출("출력", [(_,없음)])
```

### 단계 4: 의미 분석

```
입력: AST

처리:
1. 심볼 테이블 구축:
   전역스코프:
     "구하기" → 함수 { 매개변수: [(정수,"번째",의)], 반환: 정수 }
     "시작하기" → 함수 { 매개변수: [], 반환: 공허 }
     "출력" → 외부함수 { 매개변수: [(문자열,_,를)], 반환: 공허 }

   "구하기" 스코프:
     "번째" → 지역변수 { 타입: 정수, 역할: 의 }

   "시작하기" 스코프:
     "번호" → 지역변수 { 타입: 정수 }

2. 타입 검사:
   - "번째가 1보다 작거나같으면": 정수 비교 정수 → 참거짓 ✓
   - "번째에서 1을 빼서": 정수 - 정수 → 정수 ✓
   - "피보나치를 구하여": 정수 → 구하기(정수) → 정수 ✓
   - "합하여": 정수, 정수 → 정수 ✓

3. 조사-매개변수 바인딩:
   "번째에서 1을 빼서"
    → 빼기 함수의 매개변수 매칭:
      "에서" 역할 → 원본 매개변수
      "을" 역할 → 빼는값 매개변수

4. 파이프라인 타입 체이닝:
   빼기(정수,정수) → 정수  -서→
   구하기(정수) → 정수     -고→
   빼기(정수,정수) → 정수  -서→
   구하기(정수) → 정수     -여→
   합하기(정수,정수) → 정수  ✓

출력: 타입 확정 AST + 바인딩 정보
```

### 단계 5: 글IR 생성

```
입력: 타입 확정 AST

출력:

함수 @구하기(%번째: i32) → i32:
  진입:
    %조건 = 비교_작거나같음 %번째, 1
    조건분기 %조건, 기본경우, 재귀경우

  기본경우:
    반환 %번째

  재귀경우:
    %임시1 = 빼기 %번째, 1
    %임시2 = 호출 @구하기(%임시1)
    %임시3 = 빼기 %번째, 2
    %임시4 = 호출 @구하기(%임시3)
    %임시5 = 더하기 %임시2, %임시4
    반환 %임시5

함수 @시작하기() → void:
  진입:
    %번호 = 할당 i32
    저장 0, %번호
    분기 반복조건

  반복조건:
    %현재 = 적재 %번호
    %조건 = 비교_작거나같음 %현재, 9
    조건분기 %조건, 반복본문, 반복끝

  반복본문:
    %현재2 = 적재 %번호
    %결과 = 호출 @구하기(%현재2)
    호출 @글_정수출력(%결과)
    %다음 = 더하기 %현재2, 1
    저장 %다음, %번호
    분기 반복조건

  반복끝:
    반환
```

### 단계 6: LLVM IR 생성

```llvm
; 글 컴파일러 출력 - LLVM IR
; 소스: 피보나치.글

target triple = "x86_64-unknown-linux-gnu"

declare i32 @글_정수출력(i32)

define i32 @구하기(i32 %번째) !dbg !10 {
entry:
  %cmp = icmp sle i32 %번째, 1, !dbg !11
  br i1 %cmp, label %base, label %recurse

base:                                       ; 기본 경우
  ret i32 %번째, !dbg !12

recurse:                                    ; 재귀 경우
  %sub1 = sub i32 %번째, 1, !dbg !13
  %fib1 = call i32 @구하기(i32 %sub1), !dbg !13
  %sub2 = sub i32 %번째, 2, !dbg !14
  %fib2 = call i32 @구하기(i32 %sub2), !dbg !14
  %sum = add i32 %fib1, %fib2, !dbg !15
  ret i32 %sum, !dbg !15
}

define void @시작하기() !dbg !20 {
entry:
  %번호 = alloca i32, !dbg !21
  store i32 0, ptr %번호, !dbg !21
  br label %loop.cond

loop.cond:
  %cur = load i32, ptr %번호, !dbg !22
  %cmp = icmp sle i32 %cur, 9, !dbg !22
  br i1 %cmp, label %loop.body, label %loop.end

loop.body:
  %cur2 = load i32, ptr %번호, !dbg !23
  %result = call i32 @구하기(i32 %cur2), !dbg !23
  call i32 @글_정수출력(i32 %result), !dbg !23
  %next = add i32 %cur2, 1, !dbg !24
  store i32 %next, ptr %번호, !dbg !24
  br label %loop.cond

loop.end:
  ret void
}

; 진입점: _start가 시작하기를 호출
define void @_start() {
  call void @시작하기()
  call void @글_종료(i32 0)
  unreachable
}

; 디버그 메타데이터 (DWARF)

!12 = !DILocation(line: 5, column: 9, scope: !10)
!13 = !DILocation(line: 6, column: 5, scope: !10)
!14 = !DILocation(line: 7, column: 5, scope: !10)
!15 = !DILocation(line: 8, column: 5, scope: !10)
!16 = !DISubroutineType(types: !17)
!17 = !{!18, !18}
!18 = !DIBasicType(name: "정수", size: 32, encoding: DW_ATE_signed)

!21 = !DILocation(line: 11, column: 5, scope: !20)

### 단계 7-9: LLVM 최적화 → 기계어 → 링크

```
단계 7: LLVM 최적화 패스
  - mem2reg: alloca를 SSA 레지스터로 승격
  - 인라이닝: 작은 함수 인라인 확장
  - 상수 접기, 죽은 코드 제거 등

단계 8: 코드 생성 (LLVM → x86-64 어셈블리)
  구하기:
    push rbp
    mov rbp, rsp
    cmp edi, 1
    jle .base
    push rbx
    mov ebx, edi
    lea edi, [rbx - 1]
    call 구하기
    mov ecx, eax
    lea edi, [rbx - 2]
    push rcx
    call 구하기
    pop rcx
    add eax, ecx
    pop rbx
    pop rbp
    ret
  .base:
    mov eax, edi
    pop rbp
    ret

단계 9: 어셈블 → 오브젝트 파일 (LLVM MC)
  - ELF 형식 오브젝트 파일 생성 (피보나치.o)
  - .text 섹션: 기계어 코드
  - .rodata 섹션: 상수 데이터
  - .debug_info: DWARF 디버그 정보
  - 심볼 테이블: 구하기, 시작하기, _start 등
  - 재배치 테이블: 글_정수출력, 글_종료 등 외부 참조

단계 10: 링크 (LLD)
  입력: 피보나치.o + 글런타임.a (런타임 라이브러리)

  처리:
  1. 심볼 해석: 구하기 → 피보나치.o의 오프셋 0x100
  2. 글_정수출력 → 글런타임.a의 입출력.o에서 해석
  3. 글_종료 → 글런타임.a의 체계.o에서 해석
  4. 재배치 적용: call 명령어의 주소를 실제 주소로 패치
  5. ELF 헤더 생성: 진입점 = _start 주소

  출력: 피보나치 (실행 가능한 ELF 바이너리)
