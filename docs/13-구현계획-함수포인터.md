# P2.1: 함수 포인터 구현 계획

## 개요

컴파일러 자체호스팅을 위해 가장 먼저 필요한 기능.
콜백, 함수 테이블, 전략 패턴 등에 사용된다.

## 문법 설계

### 함수 포인터 타입 선언

```글
(* 방법 1: 별칭으로 함수 포인터 타입 정의 *)
별칭 비교함수는 [정수, 정수 → 정수].

(* 방법 2: 직접 변수 선언에 사용 *)
[정수, 정수 → 정수] 비교자는 없음이다.
```

### 함수 포인터 대입

```글
(* 함수 이름을 값으로 사용 → 함수 주소 취득 *)
비교자는 오름차순이다.
```

### 함수 포인터를 통한 호출

```글
(* 직접 호출 구문 사용 *)
정수 결과는 비교자(3, 5)다.
```

## C 코드 생성 목표

```c
/* 별칭 비교함수는 [정수, 정수 → 정수]. */
typedef int (*비교함수)(int, int);

/* [정수, 정수 → 정수] 비교자는 없음이다. */
int (*비교자)(int, int) = NULL;

/* 비교자는 오름차순이다. */
비교자 = 오름차순;

/* 정수 결과는 비교자(3, 5)다. */
int 결과 = 비교자(3, 5);
```

## 구현 단계

### 1단계: 토큰/렉서 수정

변경 불필요. 기존 토큰으로 충분:
- `[`, `]` — 대괄호
- `→` — 화살표
- `,` — 쉼표

### 2단계: AST 노드 추가

```c
/* 구문트리.h에 추가 */

/* 함수 포인터 타입 노드 */
/* 노드_함수포인터타입 */
struct {
    노드 **매개변수타입들;    /* 매개변수 타입 목록 */
    int   매개변수타입_수;
    노드 *반환타입;           /* 반환 타입 */
} 함수포인터타입;
```

### 3단계: 파서 수정

`구문분석기.c`에서:
1. 타입 파싱 시 `[` 로 시작하면 함수 포인터 타입으로 파싱
2. `[타입, 타입, ... → 반환타입]` 형태 인식
3. 별칭 선언에서 함수 포인터 타입 허용

### 4단계: 코드생성기 수정

`코드생성기.c`에서:
1. 함수 포인터 타입 출력: `반환타입 (*이름)(매개변수타입들)`
2. 별칭의 함수 포인터 타입: `typedef 반환타입 (*별칭이름)(매개변수타입들)`
3. 함수 이름을 값으로 사용 시 `&` 없이 이름만 출력 (C에서 함수 이름 자체가 포인터)

## 테스트

```글
포함 "표준/입출력"

[정수 가를 정수 나에 더하기]는 → 정수
    반환 가 + 나.

[정수 가를 정수 나에서 빼기]는 → 정수
    반환 가 - 나.

별칭 연산함수는 [정수, 정수 → 정수].

[시작하기]는 → 정수
    연산함수 연산은 더하기다.
    정수 결과는 연산(3, 5)다.
    결과를 출력하다.

    연산은 빼기다.
    결과는 연산(10, 4)다.
    결과를 출력하다.

    반환 0.
```

예상 출력:
```
8
6
```
