# 조사와 어미 — 글 언어의 핵심

글 언어는 한국어 문법의 **조사**(particles)와 **어미**(verb endings)를 프로그래밍 구문 요소로 사용합니다.

## 조사 (Particles) — 인자 역할 바인딩

조사는 함수 인자의 역할을 지정합니다.

| 조사 | 역할 | 설명 | 예시 |
|------|------|------|------|
| 을/를 | 목적어 | 동작의 대상 | `사과를 먹다` |
| 이/가 | 주어 | 동작의 주체 | `번호가 크면` |
| 은/는 | 주제 | 변수/함수 선언의 주제 | `결과는 5이다` |
| 에 | 목적지 | 동작의 목적지/위치 | `3에 더하기` |
| 에서 | 출처 | 동작의 출발점 | `10에서 빼기` |
| 로/으로 | 수단 | 방법/도구/변환 | `크기로 재할당` |
| 와/과 | 동반 | 함께하는 대상 | `나와 비교` |
| 보다 | 비교 | 비교 기준 | `10보다 큰` |
| 의 | 소유 | 멤버 접근 | `객체의 필드` |

### 조사 사용 규칙

1. **받침 유무에 따라 선택**: 을/를, 이/가, 은/는, 과/와 등은 앞 글자의 받침 유무에 따라 선택
2. **함수 정의에서 매개변수 역할 지정**: `[정수 가를 정수 나에 더하기]`
3. **함수 호출에서 인자 전달**: `5를 3에 더하기다.`

### 조사 분리 (형태소 분석)

```
결과를 → "결과" + "를"
점수가 → "점수" + "가"
10보다 → 10 + "보다"
```

형태소 분석기가 단어 끝의 조사를 자동으로 분리합니다.

**밑줄 보호**: `글_길이`처럼 밑줄(`_`)이 포함된 복합 식별자는 조사 분리에서 보호됩니다. `글_길이`가 `글_길` + `이`로 잘못 분리되는 것을 방지합니다.

## 어미 (Verb Endings) — 제어 흐름

어미는 문장의 문법적 기능을 결정합니다.

| 어미 | 기능 | 설명 | 예시 |
|------|------|------|------|
| -다 | 서술/종결 | 문장(명령) 종결 | `출력하다.` |
| -면 | 조건 (if) | 조건 분기 | `크면 { ... }` |
| -동안 | 반복 (while) | 반복 실행 | `작은동안 { ... }` |
| -고 | 순차 | 순차 실행 | (미구현) |
| -서 | 인과 | 원인-결과 | (미구현) |
| -거나 | 선택 (or) | 대안 선택 | (미구현) |
| -지만 | 양보 | 예외 처리 | (미구현) |

### 비교 동사 + 어미 조합

비교 표현은 비교 동사와 어미를 조합합니다:

```
번호가 10보다 작으면          // if (번호 < 10)
점수가 90보다 크거나같으면    // if (점수 >= 90)
번호가 100보다 작은동안       // while (번호 < 100)
```

비교 동사:
| 동사 | 연산자 | 관형사형(-ㄴ) | 연결형(-으) |
|------|--------|------------|-----------|
| 작다 | < | 작은 | 작으 |
| 크다 | > | 큰 | 크 |
| 같다 | == | 같은 | 같으 |
| 다르다 | != | 다른 | 다르 |
| 작거나같다 | <= | 작거나같은 | 작거나같으 |
| 크거나같다 | >= | 크거나같은 | 크거나같으 |

## 서술격 조사 "이다"

`이다`는 한국어의 서술격 조사(copula)로, 글 언어에서 변수 대입에 사용됩니다:

```
정수 나이는 25이다.          // int 나이 = 25;
정수 결과는 5를 계산이다.    // int 결과 = 계산(5);
```

형태소 분석기는 `이다`를 `이` + `다`로 분리합니다:
- `이` — 조사 (파서가 소비)
- `다` — 어미 (문장 종결)

**주의**: 밑줄 포함 식별자 뒤에서는 `이다` 분리가 적용되지 않습니다. `글_길이다` → `글_길이` + `다` (이중분리 방지)
