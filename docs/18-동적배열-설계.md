# 18. 동적 배열 (목록) 설계

## 개요

글 언어에 동적 배열(가변 길이 배열) 기능을 추가한다. C의 `realloc` 기반 doubling 전략으로 구현하며, 한글 문법에 자연스럽게 통합한다.

## 문법

### 선언

```글
(* 타입 명시 목록 *)
정수 목록 숫자들.

(* 타입 추론 목록 — 빈 목록 *)
목록 항목들.

(* 초기값 포함 *)
정수 목록 숫자들은 {1, 2, 3}.
```

### 연산

```글
(* 원소 추가 *)
42를 숫자들에 추가하다.

(* 인덱스 접근 — 기존 배열 문법과 동일 *)
숫자들[0].

(* 마지막 원소 제거 *)
숫자들에서 제거하다.

(* 특정 인덱스 제거 *)
2에서 숫자들을 제거하다.

(* 길이 조회 *)
정수 길이는 숫자들의 길이.

(* 비우기 *)
숫자들을 비우다.
```

## 내부 표현

### 구조체

```c
typedef struct {
    void *데이터;      /* 원소 배열 포인터 */
    int 길이;          /* 현재 원소 수 */
    int 용량;          /* 할당된 용량 */
    int 원소크기;      /* sizeof(원소 타입) */
} 글_목록;
```

### 메모리 전략

- 초기 용량: 8
- 확장: 용량이 부족하면 2배로 확장 (`realloc`)
- 축소: 길이가 용량의 1/4 이하이면 1/2로 축소 (선택적)

## AST 노드

```
노드_목록선언:
    .타입       — 원소 타입 노드
    .이름       — 변수 이름
    .초기값     — 배열 리터럴 노드 (선택)
```

## C 코드 생성

### 선언

```글
정수 목록 숫자들.
```

생성 코드:

```c
글_목록 숫자들 = {NULL, 0, 0, sizeof(int)};
```

### 추가

```글
42를 숫자들에 추가하다.
```

생성 코드:

```c
글_목록_추가(&숫자들, &(int){42});
```

### 접근

```글
숫자들[0]
```

생성 코드:

```c
((int*)숫자들.데이터)[0]
```

### 길이

```글
숫자들의 길이
```

생성 코드:

```c
숫자들.길이
```

## 표준 라이브러리: 표준/목록.글

런타임 함수 (C로 구현, 글 래퍼 제공):

```글
(* 목록 초기화 *)
[참조 글_목록 목록을 정수 원소크기로 목록_초기화]는 {
    목록→데이터 = 0.
    목록→길이 = 0.
    목록→용량 = 0.
    원소크기를 목록→원소크기에 넣다.
}

(* 원소 추가 *)
[참조 글_목록 목록을 참조 공허 원소를 목록_추가]는 { ... }

(* 원소 제거 — 마지막 *)
[참조 글_목록 목록을 목록_제거]는 { ... }

(* 인덱스로 제거 *)
[참조 글_목록 목록을 정수 인덱스에서 목록_인덱스제거]는 { ... }

(* 비우기 *)
[참조 글_목록 목록을 목록_비우기]는 { ... }

(* 해제 *)
[참조 글_목록 목록을 목록_해제]는 { ... }
```

## 구현 단계

### 1단계: 파서
- `목록` 키워드 추가 (토큰 종류)
- `노드_목록선언` AST 노드 추가
- `변수선언파싱`에서 `타입 목록 이름` 패턴 인식

### 2단계: 코드 생성기
- `노드_목록선언` → `글_목록` 구조체 선언 코드 생성
- `추가하다`/`제거하다` 호출을 런타임 함수 호출로 변환
- 배열 접근 `[i]` → 타입 캐스팅 포인터 접근

### 3단계: 표준 라이브러리
- `표준/목록.글` 작성
- `글_목록` 구조체 정의
- `목록_초기화`, `목록_추가`, `목록_제거`, `목록_해제` 구현

### 4단계: 자체호스팅 반영
- 자체호스팅 컴파일러에도 동일한 변경 적용
- 부트스트랩 → 자체호스팅 순서로 구현

## 고려사항

- **타입 안전성**: 컴파일 시점에 원소 타입 확인 (타입 불일치 경고)
- **메모리 해제**: 스코프 종료 시 자동 해제는 미지원 (수동 `목록_해제` 필요)
- **중첩 목록**: `정수 목록 목록` → 2차원 동적 배열 (향후 확장)
- **반복자**: `숫자들의 각 원소를 ... 하다` 문법 (향후 확장)
