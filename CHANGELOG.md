# 변경 이력

## v0.7.2 (2026-04-14) — 7건 버그 수정: 주소 취득, 포인터 산술 스케일링, 묶음 참조 매개변수

`문제점.txt`에 나열된 v0.6.5 잔존 결함을 대대적으로 해결. 자세한 내용은 `docs/22-버그수정보고서-v0.7.2.md` 참조.

### 버그 수정

- **`&배열[인덱스]` 쓰레기 주소/크래시**: 파서 우선순위(`&`가 `기본표현식` 대신 `접미표현식` 사용) + 신규 IR opcode `명령_원소주소` 추가로 이중 해결. 자식이 `노드_배열접근`이면 값 로드 없이 `base + index*size`만 계산.
- **포인터 산술 스케일링 (`*(포인터 + N)` 바이트 단위 문제)**: 기존 `임시변수원소크기풀`이 "배열이다"와 "포인터 원소크기" 두 신호를 겸용하여 포인터에 설정 시 자체호스팅 붕괴. 전용 필드 `임시변수포인터원소크기풀` 신설로 분리 해결. 대상 타입별 크기 결정(`문자/공허` = 1, 정수/실수/묶음 = 8).
- **묶음 포인터 매개변수 `의` 접근 쓰레기값**: `pe_gen.gl`의 `명령_필드적재/저장`에서 포인터 매개변수 감지 시 자동으로 화살표 시맨틱 적용 (`종류 == 3` AND `원소크기 > 0`).
- **전역 변수 음수 초기화가 0으로 평가**: 컴파일타임 상수 접힘 헬퍼 `정수상수표현인가` / `정수상수평가` 신설. 리터럴 + 단항/이항 산술(`+`, `-`, `*`, `/`) 재귀 평가 지원.
- **숫자 상수 매크로 런타임 0**: 식별자 해석 시 매크로 테이블 조회 추가. 숫자(정수/실수) 매크로 값을 상수 리터럴로 방출.
- **형태소 분리 크래시 (`와이` → `와+이`)**: 1음절 어근이 접사 자체와 일치할 때 분리 금지. 자기-참조 심볼 등록 루프 차단.
- **문자열 보간+형식 혼용 인자 오류**: 추가 인자가 있으면 보간 경로 회피. 순수 보간/순수 형식 중 하나만 사용하는 정책.

### 내부 변경

- 신규 IR opcode: `명령_원소주소`
- 신규 임시변수 풀: `임시변수포인터원소크기풀[65536]`
- 신규 헬퍼 함수: `참조원소크기결정`, `정수상수표현인가`, `정수상수평가`

- 회귀 테스트 `tests/*.gl` 9개 전원 통과

## v0.6.3 (2026-04-07) — typedef + const + union + unsigned + 포인터 산술

## v0.6.1 (2026-04-07) — 코드 리뷰 12건 수정 + 프로젝트 경로 영문화

**코드 리뷰에서 발견된 12개 이슈를 수정하고, 한글 경로를 영문으로 전환하여 네이티브 자체호스팅 고정점을 재달성했습니다.**

### 버그 수정 (P0 크리티컬)
- **64비트 상수 정수 잘림**: `명령_상수정수` 처리에서 `긴정수` 풀의 값을 `정수`(32비트)로 받아 상위 32비트가 잘리던 문제. imm64 인코딩(49 BA + 8바이트) 분기 추가.
- **strtoll_10 32비트 오버플로**: `정수로_변환()`(atoi) 대신 수동 64비트 파싱 루프로 교체. 2^31 초과 정수 리터럴 정상 파싱.

### 버그 수정 (P1 높음)
- **묶음 필드 오프셋 충돌**: 서로 다른 묶음의 동명 필드가 같은 오프셋을 반환하던 문제. `묶음오프셋소속풀` 배열 추가로 복합 키 적용.
- **disp8 범위 조건 오류 (10곳)**: `오프셋 >= -127`을 `-128`로, `오프셋 < 0`을 `<= 127`로 수정. 양수 오프셋도 disp8 인코딩 가능.
- **파서_오류있는가() 미정의 심볼**: 존재하지 않는 함수 호출을 `오류수 > 0` 직접 비교로 교체.

### 버그 수정 (P2 중간)
- **토큰 버퍼 오버플로**: 크기 8→16 확장, 초과 시 경고 메시지 출력.
- **버퍼 오버플로 방어**: 논리/IR 레이블 버퍼, 전처리 4MB 버퍼에 경계 검사 및 치명적 오류 종료 추가.
- **심볼 테이블 로드 팩터**: 75% 초과 시 경고 메시지 출력.

### 성능 개선
- **레이블 검색 해시 테이블**: 선형 검색 O(N²) → djb2 해시 O(N) 패치 해결.

### 기타
- **ELF .rodata 권한**: 쓰기 권한(PF_W) 제거 → 읽기 전용(PF_R).
- **인라인 임시매핑 크기 제한**: 활성화 시 256 초과 확인 필요 주석 추가.

### 프로젝트 구조 변경

- 컴파일러 소스: `자체호스팅/*.글` → `src/*.gl`
- 표준 라이브러리: `표준/*.글` → `std/*.gl`
- 부트스트랩: `부트스트랩/` → `bootstrap/`

## v0.6.0 (2026-04-05) — 실수 선언 수정 + 네이티브 자체호스팅 안정화

**로컬 실수 변수 선언이 이제 완전히 동작하며, 8바이트 고정 크기 배열의 주소 취득 버그를 근본 수정했습니다.**

고정점 SHA256: `afc0c00b6c1d580f2e92bb38e9e47ec1a76551ae19c44e184cf6a4b7e83d36c0`

### 버그 수정
- **8바이트 배열 lea/mov 오판정**: 전역/로컬 배열의 주소 취득 시 `크기풀 > 8` 조건만 사용하여 `문자[8]` 같은 8바이트 고정 배열을 스칼라로 오인 → `mov reg, [rip+disp]`로 BSS 첫 8바이트(=0)를 값으로 로드 → NULL 포인터 전달 → `memcpy(NULL, ...)` crash. `임시변수원소크기풀 > 0` 조건 추가로 배열 정확히 판단.
- **실수 매개변수 스택 홈 슬롯 누락**: 함수 프롤로그에서 RCX/RDX/R8/R9만 저장하고 XMM0~XMM3을 저장하지 않아 `&실수매개변수` 주소 취득이 쓰레기 슬롯을 가리키던 문제 수정.
- **함수 호출 실수 반환 처리**: 모든 호출 반환을 `PE_저장_rax`로 저장하던 로직을 실수 반환 타입 시 `PE_저장실수_xmm0`로 분기.
- **전역 실수 변수 RIP 상대 주소**: `PE_적재실수_xmm0/xmm1`, `PE_저장실수_xmm0`에서 종류 2(전역) 경로 누락 수정 — 이전에는 전역 실수 변수를 rbp 상대로 접근해 스택 오염 유발.

### 신규 기능
- `인코딩_movsd_mem_xmmN` 헬퍼 — xmm0~xmm3 범용 실수 저장 명령

### 검증
- 네이티브 자체호스팅 4회 고정점 일치
- 실수 선언/출력, 정수 함수 호출, SOV 호출, 보간 — 모두 통과
