# 변경 이력

## v0.6.0 (2026-04-05) — 실수 선언 수정 + 네이티브 자체호스팅 안정화

**로컬 실수 변수 선언이 이제 완전히 동작하며, 8바이트 고정 크기 배열의 주소 취득 버그를 근본 수정했습니다.**

고정점 SHA256: `afc0c00b6c1d580f2e92bb38e9e47ec1a76551ae19c44e184cf6a4b7e83d36c0`

### 버그 수정
- **8바이트 배열 lea/mov 오판정**: 전역/로컬 배열의 주소 취득 시 `크기풀 > 8` 조건만 사용하여 `문자[8]` 같은 8바이트 고정 배열을 스칼라로 오인 → `mov reg, [rip+disp]`로 BSS 첫 8바이트(=0)를 값으로 로드 → NULL 포인터 전달 → `memcpy(NULL, ...)` crash. `임시변수원소크기풀 > 0` 조건 추가로 배열 정확히 판단.
- **실수 매개변수 스택 홈 슬롯 누락**: 함수 프롤로그에서 RCX/RDX/R8/R9만 저장하고 XMM0~XMM3을 저장하지 않아 `&실수매개변수` 주소 취득이 쓰레기 슬롯을 가리키던 문제 수정.
- **함수 호출 실수 반환 처리**: 모든 호출 반환을 `PE_저장_rax`로 저장하던 로직을 실수 반환 타입 시 `PE_저장실수_xmm0`로 분기.
- **전역 실수 변수 RIP 상대 주소**: `PE_적재실수_xmm0/xmm1`, `PE_저장실수_xmm0`에서 종류 2(전역) 경로 누락 수정 — 이전에는 전역 실수 변수를 rbp 상대로 접근해 스택 오염 유발.

### 신규 기능
- `인코딩_movsd_mem_xmmN` 헬퍼 — xmm0~xmm3 범용 실수 저장 명령

### 검증
- 네이티브 자체호스팅 4회 고정점 일치
- 실수 선언/출력, 정수 함수 호출, SOV 호출, 보간 — 모두 통과

## v0.5.0 (2026-04-02) — 네이티브 자체호스팅

**네이티브 컴파일러가 자기 자신을 컴파일하는 고정점에 도달했습니다.**

고정점 SHA256: `970956052d2a61cd9fb1809783b71484fdfb8eabebfa761a6002840e8f4ee415`

### 신규 기능
- 네이티브 자체호스팅 달성 (stage2 == stage3)
- fopen UTF-8/ACP 폴백 래퍼 — 한글 경로 파일 열기 지원
- `_atoi64` 도입 — 64비트 정수 리터럴 파싱
- 미해결 패치 경고 시스템

### 버그 수정
- **strndup 호출 미패치**: 사용자 정의 함수를 C 표준함수로 잘못 분류 → call displacement 0
- **5번째+ 매개변수 스택 오프셋**: callee-saved 레지스터 push 후 오프셋 미조정 → 리턴 주소를 매개변수로 읽음
- **BSS 섹션 플래그 오버플로**: 0xC0000080 > INT_MAX → atoi가 0x7FFFFFFF 반환 → BSS 쓰기 권한 없음

### 성능
- 컴파일 속도: 글cc(C 트랜스파일) 대비 **77배 빠름**
- 실행 성능: C /Od 대비 약 2.2배, Python 대비 약 9.4배
